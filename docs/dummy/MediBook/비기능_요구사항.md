# MediBook — Non-Functional Requirements

## 1. Performance

### 1.1 API Response Time

All search APIs (PRD 3.1 clinic search, PRD 3.2 doctor profile) must respond within 500ms at the 95th percentile under normal load. The symptom recommendation API (PRD 3.3) may take up to 3 seconds due to ML model inference.

### 1.2 Concurrent Users

The system must support at least 1,000 concurrent users performing appointment bookings (PRD 4.1) without degradation. Peak load is expected during morning hours (9:00-10:00 AM) when patients typically book appointments.

### 1.3 Notification Delivery

SMS reminders (PRD 4.3) must be delivered within 60 seconds of the scheduled send time. Appointment confirmation notifications (PRD 4.1) must be sent within 30 seconds of booking completion.

## 2. Availability

### 2.1 Uptime

The appointment booking system (PRD 4.1, 4.2) requires 99.9% uptime. Search and browsing features (PRD 3.1, 3.2, 3.3) can tolerate 99.5% uptime. Scheduled maintenance windows are permitted between 02:00-04:00 AM KST.

### 2.2 Data Backup

Patient profiles (PRD 2.2) and appointment data (PRD 4.1) must be backed up every 6 hours. Medical records (PRD 5.1) received from healthcare institutions must be backed up immediately upon receipt. Backup retention: 90 days.

## 3. Security

### 3.1 Authentication

All API endpoints except public search (PRD 3.1, 3.3) require JWT authentication (PRD 2.2). Token expiration and refresh policies are defined in the API Specification (Section 1.2). Failed login attempts are logged for security monitoring.

### 3.2 Data Encryption

Health information (PRD 2.2 — allergies, medications, conditions) must be encrypted at rest using AES-256 as specified in the Privacy Policy (Section 5). All API communication must use TLS 1.3.

### 3.3 Audit Logging

All access to patient health information (PRD 2.2) and medical records (PRD 5.1) must be logged with timestamp, accessor identity, and action performed. Audit logs are retained for 3 years per healthcare compliance requirements.

## 4. Accessibility

### 4.1 Mobile Responsiveness

All patient-facing features (search, booking, profile management) must be fully functional on mobile devices with screen width >= 320px. The appointment slot selection UI (PRD 4.1) must be touch-friendly with minimum tap targets of 44x44px.

### 4.2 Language Support

MVP supports Korean only. UI text must be externalized for future internationalization. Medical terminology in symptom recommendations (PRD 3.3) must include plain-language descriptions alongside technical terms.
