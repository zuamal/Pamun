# LearnHub — Notification Specification

This document specifies all automated notifications in the LearnHub system. Each notification references the originating PRD requirement and the API endpoint that triggers it.

## 1. Enrollment Notifications

### 1.1 Course Assignment

**Trigger:** Admin assigns learner to a course (PRD 3.3, API 2.3 POST /api/courses/{id}/enrollments).

**Channel:** Email

**Content:**
- Subject: "[LearnHub] You have been assigned to: {course_title}"
- Body: Course title, description, deadline (enrollment.deadline from Data Model 3.1), link to course page
- Sent to: Each assigned learner

### 1.2 Deadline Reminders

**Trigger:** Scheduled batch job checks enrollment deadlines (PRD 3.3, API 2.3).

**Channel:** Email

**Schedule:**
- 7 days before deadline
- 3 days before deadline
- 1 day before deadline (urgent tone)

**Content:**
- Subject: "[LearnHub] Reminder: {course_title} deadline in {days} days"
- Body: Current progress_percent (computed from Data Model 3.3), remaining lessons count, deadline date, link to resume

**Suppression:** Do not send if enrollment.status = COMPLETED.

## 2. Progress Notifications

### 2.1 Behind Schedule Alert

**Trigger:** Batch job computes is_behind_schedule = true (Data Model 3.3, PRD 4.2).

**Channel:** Email

**Schedule:** Weekly (every Monday 9:00 AM), only if still behind.

**Content:**
- Subject: "[LearnHub] You are falling behind on: {course_title}"
- Body: Expected progress vs actual progress, days remaining, link to course

**Logic:** is_behind_schedule = true when (elapsed_time / total_time) > (completed_lessons / total_lessons) + 0.1 buffer. This formula ensures alerts are not overly aggressive and account for front-loaded or back-loaded study patterns.

### 2.2 Module Completion

**Trigger:** Learner completes last lesson in a module (all LessonProgress.is_completed = true for module, Data Model 3.2).

**Channel:** In-app notification only (no email).

**Content:** "Module '{module_title}' completed! Next: {next_module_title}" or "All modules completed! Take the quiz to earn your certificate." if last module.

## 3. Assessment Notifications

### 3.1 Quiz Available

**Trigger:** Learner completes module/course that has an associated quiz (PRD 5.2, API 4.2).

**Channel:** Email + In-app

**Content:**
- Subject: "[LearnHub] Quiz available: {quiz_title}"
- Body: Quiz title, associated course/module, max_attempts (from Data Model 4.1), link to quiz

### 3.2 Grading Complete (Subjective)

**Trigger:** Instructor completes manual grading (API 4.2 PATCH /api/quiz-attempts/{id}/grade, PRD 5.2).

**Channel:** Email + In-app

**Content:**
- Subject: "[LearnHub] Your quiz has been graded: {quiz_title}"
- Body: Total score, pass/fail status, link to results. If failed: "You have {remaining_attempts} attempts remaining." (computed from max_attempts - attempt_number, Data Model 4.3).

### 3.3 Certificate Issued

**Trigger:** Certificate auto-generated upon course completion (PRD 5.3, API 4.3, Data Model 5.1).

**Channel:** Email + In-app

**Content:**
- Subject: "[LearnHub] Congratulations! Certificate earned: {course_title}"
- Body: Certificate number, link to download PDF, course completion summary

## 4. Administrative Notifications

### 4.1 Pending Grading Alert

**Trigger:** Subjective quiz attempt submitted, grading not completed within 3 days (Data Model 4.3: submitted_at vs current time).

**Channel:** Email to course instructor

**Content:**
- Subject: "[LearnHub] Pending grading: {quiz_title} ({count} submissions)"
- Body: Number of ungraded submissions, learner names, link to grading page

### 4.2 Low Completion Rate Alert

**Trigger:** Course completion rate falls below 50% with less than 30% of enrollment period remaining (PRD 6.1).

**Channel:** Email to course instructor + Admin

**Content:**
- Subject: "[LearnHub] Low completion alert: {course_title}"
- Body: Current completion rate, number of non-starters (Data Model 3.1 where no LessonProgress exists), deadline distribution

## 5. Notification Preferences

Learners can configure notification preferences per channel:

| Notification Type | Email | In-app | Can Disable? |
|------------------|-------|--------|-------------|
| Course Assignment | Yes | Yes | No (mandatory) |
| Deadline Reminders | Yes | Yes | Yes |
| Behind Schedule | Yes | No | Yes |
| Module Completion | No | Yes | Yes |
| Quiz Available | Yes | Yes | No (mandatory) |
| Grading Complete | Yes | Yes | No (mandatory) |
| Certificate Issued | Yes | Yes | No (mandatory) |

Mandatory notifications cannot be disabled as they are essential for the learning workflow. Preference settings are stored per user and accessed via GET/PATCH /api/users/me/notification-preferences.
