# LearnHub — Data Model Specification

This document defines the core data models for LearnHub LMS. All models correspond to PRD requirements and are referenced by the API specification and test plan.

## 1. User Models

### 1.1 User

Represents a registered employee in the system (PRD 2.1, 2.2).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| employee_id | string | Employee number from SSO (PRD 2.2) |
| name | string | Full name (synced from SSO) |
| department | string | Department name (synced from SSO) |
| position | string | Job title (synced from SSO) |
| role | enum | ADMIN, INSTRUCTOR, LEARNER (default: LEARNER, PRD 2.1) |
| email | string | Corporate email |
| created_at | datetime | First SSO login timestamp |

### 1.2 Role Permissions

Role-based access control mapping (PRD 2.1).

| Permission | Admin | Instructor | Learner |
|-----------|-------|-----------|---------|
| Create/edit courses | Yes | Own courses only | No |
| Assign learners | Yes | No | No |
| Create quizzes | Yes | Own courses only | No |
| Grade subjective answers | Yes | Own courses only | No |
| View all learner progress | Yes | Own courses only | Own only |
| Download reports | Yes | Own courses only | Own only |
| Manage user roles | Yes | No | No |

## 2. Course Models

### 2.1 Course

Represents an educational course (PRD 3.1).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| title | string | Course title |
| description | text | Course description |
| category | string | Category (e.g., Compliance, Technical, Leadership) |
| type | enum | PUBLIC (all employees), ASSIGNED (specific learners, PRD 3.1) |
| instructor_id | UUID | FK to User (role=INSTRUCTOR) |
| enrollment_period_days | int | Duration for completion after assignment (PRD 3.3) |
| sequential_mode | boolean | If true, modules must be completed in order (PRD 3.2) |
| passing_score | int | Minimum quiz score for certification (PRD 5.1) |
| status | enum | DRAFT, PUBLISHED, ARCHIVED |
| created_at | datetime | |
| updated_at | datetime | |

### 2.2 Module

A section within a course (PRD 3.2).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| course_id | UUID | FK to Course |
| title | string | Module title |
| order_index | int | Display order within course |
| created_at | datetime | |

### 2.3 Lesson

Individual learning unit within a module (PRD 3.2).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| module_id | UUID | FK to Module |
| title | string | Lesson title |
| type | enum | VIDEO, DOCUMENT, TEXT |
| content_url | string | URL for video or PDF (null for TEXT type) |
| text_content | text | Rich text content (null for VIDEO/DOCUMENT type) |
| duration_seconds | int | Video duration (null for non-video) |
| order_index | int | Display order within module |

## 3. Enrollment and Progress Models

### 3.1 Enrollment

Links a learner to a course (PRD 3.3).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| user_id | UUID | FK to User (learner) |
| course_id | UUID | FK to Course |
| assigned_by | UUID | FK to User (admin who assigned, null for public enrollment) |
| assigned_at | datetime | When the learner was assigned |
| deadline | datetime | Enrollment period end (assigned_at + course.enrollment_period_days) |
| completed_at | datetime | Null until all modules + quiz completed |
| status | enum | ACTIVE, COMPLETED, EXPIRED |

### 3.2 LessonProgress

Tracks individual lesson completion (PRD 4.1).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| user_id | UUID | FK to User |
| lesson_id | UUID | FK to Lesson |
| is_completed | boolean | Completion status |
| video_watched_percent | float | For VIDEO lessons, must reach 90% (PRD 4.1) |
| last_position_seconds | int | Resume position for video (PRD 4.1) |
| completed_at | datetime | Null until completed |
| last_accessed_at | datetime | For resume functionality |

### 3.3 CourseProgress (Computed)

Not stored directly but computed from LessonProgress (PRD 4.2).

| Field | Type | Description |
|-------|------|-------------|
| user_id | UUID | |
| course_id | UUID | |
| completed_lessons | int | Count of completed lessons |
| total_lessons | int | Total lessons in course |
| progress_percent | float | completed_lessons / total_lessons * 100 |
| is_behind_schedule | boolean | Progress rate < expected rate based on deadline |

## 4. Assessment Models

### 4.1 Quiz

Assessment for a module or course (PRD 5.1).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| course_id | UUID | FK to Course |
| module_id | UUID | FK to Module (null if course-level quiz) |
| title | string | Quiz title |
| max_attempts | int | Maximum number of attempts (PRD 5.2), null for unlimited |
| created_by | UUID | FK to User (instructor) |

### 4.2 Question

Individual quiz question (PRD 5.1).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| quiz_id | UUID | FK to Quiz |
| type | enum | SINGLE_CHOICE, MULTI_CHOICE, SHORT_ANSWER |
| content | text | Question text |
| options | json | Array of options (null for SHORT_ANSWER) |
| correct_answer | json | Correct option(s) or expected answer |
| points | int | Score for this question |
| order_index | int | Display order |

### 4.3 QuizAttempt

Records a learner's quiz submission (PRD 5.2).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| user_id | UUID | FK to User |
| quiz_id | UUID | FK to Quiz |
| attempt_number | int | Which attempt (1, 2, ...) |
| answers | json | Submitted answers |
| auto_score | int | Auto-graded score (objective questions) |
| manual_score | int | Instructor-graded score (subjective, null until graded) |
| total_score | int | auto_score + manual_score |
| passed | boolean | total_score >= course.passing_score |
| submitted_at | datetime | |
| graded_at | datetime | Null until fully graded |

## 5. Certificate Model

### 5.1 Certificate

Auto-issued upon course completion (PRD 5.3).

| Field | Type | Description |
|-------|------|-------------|
| id | UUID | Primary key |
| user_id | UUID | FK to User |
| course_id | UUID | FK to Course |
| certificate_number | string | Unique verification code |
| issued_at | datetime | |
| pdf_url | string | Generated PDF download URL |

Certificate issuance conditions (PRD 5.3): all modules completed AND quiz total_score >= course.passing_score. Certificate PDF includes: learner name, course title, completion date, certificate number.
