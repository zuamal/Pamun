# F22 — 온보딩 투어 (코치마크)

## 개요

4단계 워크플로우를 처음 접하는 사용자에게 각 페이지의 핵심 UI 요소를 spotlight + 툴팁으로 안내하는 코치마크 투어를 추가한다.

| 환경 | 트리거 |
|------|--------|
| 데모 모드 | 각 페이지 최초 진입 시 자동 시작 (세션당 1회) |
| 셀프호스팅 | Sidebar 하단 "가이드 보기" 버튼 클릭 |

---

## 완료 기준

### 패키지 설치

- [ ] `react-joyride` 설치: `npm install react-joyride`

### 투어 스텝 정의 (`src/lib/tourSteps.ts`)

- [ ] 아래 8스텝을 `Step[]` 배열로 정의. 각 스텝의 `target`은 해당 요소에 추가할 CSS class selector(`[data-tour="..."]` 방식 권장):

| 스텝 | 페이지 | target | content |
|------|--------|--------|---------|
| 1 | UploadPage | `[data-tour="demo-btn"]` | 샘플 번들을 선택하면 파싱·추론이 완료된 세션을 즉시 불러옵니다 |
| 2 | UploadPage | `[data-tour="parse-btn"]` | 클릭하면 요구사항 추출 결과를 확인할 수 있습니다 |
| 3 | ReviewPage | `[data-tour="req-list"]` | LLM이 추출한 요구사항 목록입니다. 항목을 클릭하면 수정·삭제·병합할 수 있습니다 |
| 4 | ReviewPage | `[data-tour="infer-btn"]` | 클릭하면 요구사항 간 의존관계를 추론합니다 |
| 5 | GraphPage | `[data-tour="graph-area"]` | 승인된 연결로 구성된 의존관계 그래프입니다. 노드 핸들을 드래그하면 새 연결을 추가할 수 있습니다 |
| 6 | GraphPage | `[data-tour="edge-panel"]` | PENDING 연결을 승인 또는 거부하세요 |
| 7 | ImpactPage | `[data-tour="changed-toggle"]` | 변경 예정 요구사항에 토글을 켜면 영향 분석이 자동 실행됩니다 |
| 8 | ImpactPage | `[data-tour="impact-result"]` | 항목을 클릭하면 원문에서 해당 위치를 확인합니다 |

### 투어 상태 관리 (`src/stores/tourStore.ts`)

- [ ] Zustand store 신규 생성:
  ```ts
  interface TourStore {
    isRunning: boolean
    stepIndex: number
    setRunning: (v: boolean) => void
    setStepIndex: (i: number) => void
  }
  ```
- [ ] `sessionStorage` 플래그 키: `pamun_tour_seen_{pageName}` (예: `pamun_tour_seen_upload`)

### 각 페이지에 Joyride 컴포넌트 추가

- [ ] `UploadPage`, `ReviewPage`, `GraphPage`, `ImpactPage` 각각에 해당 페이지 스텝만 포함한 `<Joyride>` 인스턴스를 추가
- [ ] 데모 모드(`isDemoMode()`) + 해당 페이지 `sessionStorage` 플래그 미설정 시 `run={true}`로 자동 시작
- [ ] 투어 종료(`status === 'finished' || 'skipped'`) 시 `sessionStorage` 플래그 설정
- [ ] Joyride 기본 스타일을 프로젝트 색상(slate, blue)에 맞게 커스터마이징:
  ```ts
  styles={{
    options: {
      primaryColor: '#3b82f6',  // blue-500
      zIndex: 10000,
    }
  }}
  ```

### 각 페이지 target 요소에 data 속성 추가

- [ ] `UploadPage.tsx`: "데모 체험" 버튼에 `data-tour="demo-btn"`, "파싱 시작" 버튼에 `data-tour="parse-btn"`
- [ ] `ReviewPage.tsx`: 요구사항 목록 컨테이너에 `data-tour="req-list"`, "의존관계 추론 시작" 버튼에 `data-tour="infer-btn"`
- [ ] `GraphPage.tsx`: 그래프 영역 div에 `data-tour="graph-area"`, Edge 검토 패널에 `data-tour="edge-panel"`
- [ ] `ImpactPage.tsx`: changed 토글 영역에 `data-tour="changed-toggle"`, 영향 결과 패널에 `data-tour="impact-result"`

### Sidebar에 "가이드 보기" 버튼 추가 (`Stepper.tsx` 또는 `AppShell.tsx`)

- [ ] Stepper 하단에 "가이드 보기" 버튼 추가
- [ ] 클릭 시 현재 경로에 해당하는 페이지의 `sessionStorage` 플래그를 초기화하고 `tourStore.setRunning(true)` 호출
- [ ] 버튼 스타일: Sidebar 배경에 어울리는 작은 텍스트 버튼 (예: `text-xs text-slate-400 hover:text-slate-600`)

### 타입 체크

- [ ] `npx tsc --noEmit` 오류 없음
