# F5 — Edge Inference & Management

## 개요

LLM으로 요구사항 간 의존관계를 추론하고, PM이 추론 결과를 검토·수정하거나 수동으로 Edge를 추가할 수 있는 API를 구현한다.

## 완료 기준

- [ ] `POST /api/edges/infer` — 현재 스토어의 요구사항을 기반으로 LLM 의존관계 추론
  - `requirement_ids` 지정 시 해당 요구사항만 대상. `null`이면 전체 요구사항 대상
  - 추론된 Edge는 `status=PENDING`으로 스토어에 저장
  - 이미 존재하는 동일 source-target 쌍의 Edge는 중복 생성하지 않음 (스킵)
  - 스토어에 요구사항이 없으면 422
  - 응답: 새로 생성된 `Edge` 목록
- [ ] `GET /api/edges` — Edge 목록 반환
  - `?status=pending|approved|rejected` 쿼리 파라미터로 필터링 가능
  - 파라미터 없으면 전체 반환
- [ ] `POST /api/edges` — PM 수동 Edge 추가
  - `status`는 항상 `APPROVED`로 자동 설정 (요청 바디에 status 필드 없음)
  - `source_id` 또는 `target_id`가 존재하지 않는 요구사항 ID면 422
  - 동일 source-target 쌍이 이미 존재하면 422
  - 생성된 `Edge` 반환
- [ ] `PATCH /api/edges/{id}` — Edge 수정
  - `status`, `relation_type`, `evidence` 수정 가능
  - 없는 ID 요청 시 404
  - 수정된 `Edge` 반환
- [ ] `DELETE /api/edges/{id}` — Edge 삭제
  - 없는 ID 요청 시 404
- [ ] `APPROVED` 상태로 변경 시 networkx 그래프에 즉시 반영된다
- [ ] `REJECTED` 또는 삭제 시 networkx 그래프에서 즉시 제거된다
- [ ] `cd backend && mypy app/ --strict` 오류 없음
- [ ] `pytest tests/test_edges.py -v` 통과:
  - Infer: LLM mock, 중복 스킵, 요구사항 없음 오류
  - Manual create: 정상, 없는 requirement ID, 중복 케이스
  - PATCH: status 변경 + 그래프 반영 확인
  - DELETE: 삭제 + 그래프 반영 확인

## 구현 범위

```
backend/
├── app/
│   ├── api/
│   │   └── edges.py          # 5개 엔드포인트
│   ├── llm/
│   │   └── inferrer.py       # infer_edges(requirements) → list[Edge]
│   └── services/
│       └── edge_service.py   # Edge CRUD + 그래프 동기화
├── tests/
│   └── test_edges.py
```

**LLM 추론 프롬프트 방향:**
- 요구사항 목록을 인덱스와 함께 제공 (index → ID 매핑은 backend에서 처리)
- `InferenceResult` 스키마 사용 (`source_index`, `target_index` 기반)
- `InferredEdge.source_index` / `target_index` → 실제 requirement ID로 변환

**Edge ID 생성:** `uuid4()` 사용.
