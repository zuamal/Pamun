# F19 — Frontend-only Demo Mode + GitHub Pages 배포

## 개요

F18(백엔드 기반 데모 로드)이 완료된 상태에서, 백엔드 없이 GitHub Pages 정적 호스팅만으로 동작하는 데모 경로를 추가한다.

**두 데모 경로의 공존:**

| 경로 | 조건 | 비고 |
|------|------|------|
| F18 경로 | 백엔드 실행 중 (`VITE_DEMO_MODE` 미설정) | 셀프호스팅 사용자용 |
| F19 경로 | `VITE_DEMO_MODE=true` 빌드 | GitHub Pages 배포용 |

`VITE_DEMO_MODE=true` 빌드에서는 백엔드 API를 호출하지 않는다. "데모 체험" 버튼이 F18의 `POST /api/demo/load` 대신 정적 JSON fetch + mock API 레이어로 라우팅된다.

  ```ts

    if (isDemoMode()) return demoApi.patchRequirement(id, data)
  ```

- [ ] `isDemoMode()` 헬퍼: `import.meta.env.VITE_DEMO_MODE === 'true'` 또는 `demoStore.isDemoMode`

### GitHub Actions 워크플로우

- [ ] `.github/workflows/deploy.yml` 작성:
  ```yaml
  name: Deploy Demo to GitHub Pages
  on:
    push:
      branches: [main]
  jobs:
    build-and-deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with: { node-version: '20' }
        - run: cd frontend && npm ci
        - run: cd frontend && npm run build:demo
        - uses: peaceiris/actions-gh-pages@v4
          with:
            github_token: ${{ secrets.GITHUB_TOKEN }}
            publish_dir: frontend/dist
  ```
- [ ] main push 시 워크플로우가 정상 실행된다
- [ ] 빌드 실패 시 배포되지 않는다 (Actions 기본 동작)
- [ ] 배포 후 `{user}.github.io/pamun`에서 앱이 정상 로드된다

## F18과의 관계

F18에서 구현한 `POST /api/demo/load`, `GET /api/demo/bundles`, `DemoBundleModal`은 그대로 유지된다. F19는 `VITE_DEMO_MODE=true` 빌드에서만 활성화되는 별도 경로를 추가한다. 코드 분기는 `isDemoMode()` 헬퍼 한 곳에서 관리하므로 중복이 최소화된다.
