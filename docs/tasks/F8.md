# F8 — Frontend Foundation

## 개요

Vite + React + TypeScript 프로젝트를 설정하고, Zustand 스토어 구조와 타입 생성 파이프라인을 구축한다. 이후 모든 frontend feature(F9–F11)의 기반이 된다.

## 완료 기준

- [ ] `npm run dev` 실행 시 `http://localhost:5173`에서 앱이 기동된다
- [ ] `npx tsc --noEmit` 오류 없음
- [ ] `scripts/generate-types.sh` 실행 시:
  - `frontend/src/api/openapi.json` 생성
  - `frontend/src/api/types.generated.ts` 생성
  - 스크립트 오류 없이 완료
- [ ] `frontend/src/api/types.generated.ts`가 존재하고 backend 모델이 반영되어 있다
- [ ] 3개 Zustand 스토어 파일이 기본 구조로 존재한다:
  - `documentStore.ts`: `documents`, `setDocuments` 기본 슬라이스
  - `graphStore.ts`: `requirements`, `edges`, `setRequirements`, `setEdges`, `zoom`, `layout` 기본 슬라이스
  - `impactStore.ts`: `impactResult`, `setImpactResult` 기본 슬라이스
- [ ] `frontend/src/api/` 아래 fetch 함수 파일 4개가 타입만 잡힌 stub으로 존재한다:
  - `documents.ts`
  - `requirements.ts`
  - `edges.ts`
  - `impact.ts`
- [ ] `@xyflow/react`, `zustand`, `axios` (또는 fetch wrapper) 패키지 설치 완료

## 구현 범위

```
frontend/
├── src/
│   ├── api/
│   │   ├── openapi.json              # generate-types.sh로 생성
│   │   ├── types.generated.ts        # generate-types.sh로 생성
│   │   ├── documents.ts              # stub
│   │   ├── requirements.ts           # stub
│   │   ├── edges.ts                  # stub
│   │   └── impact.ts                 # stub
│   ├── stores/
│   │   ├── documentStore.ts
│   │   ├── graphStore.ts
│   │   └── impactStore.ts
│   ├── components/                   # 빈 디렉토리 (F9–F11에서 채움)
│   ├── pages/                        # 빈 디렉토리
│   ├── App.tsx                       # 기본 라우터 구조 (react-router-dom)
│   └── main.tsx
├── package.json
├── tsconfig.json
├── vite.config.ts                    # /api proxy → localhost:8000
└── index.html

scripts/
└── generate-types.sh                 # ADR-2 파이프라인 구현
```

**API 베이스 URL:** Vite dev proxy를 통해 `/api` → `http://localhost:8000/api` 프록시. fetch 함수에서 `/api/...`로 호출.

**타입 import 규칙:** 모든 fetch 함수와 store에서 `types.generated.ts`에서 import. 수동 타입 정의 금지.
