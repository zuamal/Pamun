# F18 — Key-free Demo Mode

## 개요

`ANTHROPIC_API_KEY` 없이 pre-parsed + pre-inferred 세션을 즉시 적재하여 그래프·영향 분석까지 완전히 체험할 수 있는 데모 모드를 구현한다.

- **Backend:** API 키를 선택적으로 변경 + 데모 세션 로드 엔드포인트
- **Frontend:** UploadPage의 "데모 체험" 버튼 + 번들 선택 UI
- **데이터:** 3개 번들(BookFlow / LearnHub / MediBook)의 `demo_session.json` 생성 및 커밋

---

## 완료 기준

### Backend — API 키 선택적 처리

- [ ] `ANTHROPIC_API_KEY` 미설정 상태에서 `uvicorn app.main:app` 기동 시 오류 없음
- [ ] `/api/parse` 또는 `/api/edges/infer` 호출 시 API 키 미설정이면 **503** 반환
  ```json
  {"detail": "LLM 기능을 사용하려면 ANTHROPIC_API_KEY가 필요합니다."}
  ```
- [ ] 나머지 모든 API는 API 키 없이도 정상 동작
- [ ] `cd backend && mypy app/ --strict` 오류 없음

### Backend — 데모 로드 엔드포인트

- [ ] `GET /api/demo/bundles` — 사용 가능한 데모 번들 목록 반환
  ```json
  [
    {"name": "BookFlow",  "description": "독서 앱 기획 문서 3종", "file_count": 3},
    {"name": "LearnHub",  "description": "학습 플랫폼 기획 문서 5종", "file_count": 5},
    {"name": "MediBook",  "description": "의료 예약 서비스 기획 문서 4종", "file_count": 4}
  ]
  ```
- [ ] `POST /api/demo/load/{bundle_name}` — 번들의 `demo_session.json`을 적재
  - 존재하지 않는 번들명 → 404
  - `demo_session.json`이 없는 번들 → 503 (`"데모 세션 파일이 준비되지 않았습니다."`)
  - 현재 상태를 초기화한 뒤 세션을 적재 (기존 `session_service.load()` 재사용)
  - 응답: `{"bundle": "LearnHub", "requirements": 18, "edges": 12}`

### 데모 세션 JSON 생성

- [ ] `scripts/generate-demo-sessions.sh` 작성:
  1. 백엔드 서버가 실행 중인지 확인
  2. 각 번들에 대해:
     - `POST /api/session/reset`
     - `POST /api/dummy/load/{bundle_name}` (F15)
     - `POST /api/parse` (모든 document_id)
     - `POST /api/edges/infer`
     - 추론된 Edge 전체를 `PATCH` → `APPROVED`
     - `POST /api/session/save`
     - 저장된 파일을 `docs/dummy/{bundle_name}/demo_session.json`으로 복사
- [ ] 3개 번들의 `demo_session.json`이 레포에 커밋된다 (`.gitignore`의 `*.session.json` 예외 처리 필요)
- [ ] 각 `demo_session.json`의 모든 Edge가 `APPROVED` 상태임을 확인

### Frontend — 데모 체험 버튼

- [ ] UploadPage 상단 또는 드롭존 하단에 "데모 체험" 버튼 추가
  - 실제 업로드 흐름과 시각적으로 구분되는 스타일 (예: outline 버튼, 별도 섹션)
- [ ] 버튼 클릭 시 번들 선택 모달 표시:
  - 3개 번들 카드: 이름, 설명, 포함 파일 수
  - 선택 → 확인 없이 즉시 `POST /api/demo/load/{bundle_name}` 호출 (reset이 백엔드에서 처리됨)
- [ ] 로드 중 모달 내 로딩 스피너
- [ ] 성공 시: `documentStore` + `graphStore` 갱신 → `/graph`로 이동 + Toast("LearnHub 데모를 불러왔습니다")
- [ ] 실패 시: `toastError()` + 모달 유지
- [ ] `npx tsc --noEmit` 오류 없음

## 배포 시 참고 (추후 구현)

> 현재 F18은 로컬 환경 기준으로 구현한다. 추후 라이브 URL 배포 시 아래 항목을 별도 작업으로 다룬다.
>
> - CORS origin을 배포 도메인으로 확장 (`app/main.py` CORS 설정)
> - `demo_session.json` 경로가 서버 파일시스템에 존재해야 하므로 정적 파일 서빙 또는 배포 이미지에 포함 필요
> - 백엔드 서버 배포 방식(Dockerfile, 클라우드 플랫폼) 결정 후 경로 resolve 전략 재검토
