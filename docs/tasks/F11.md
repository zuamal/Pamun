# F11 — Frontend: Impact Analysis

## 개요

PM이 그래프에서 요구사항에 "변경 예정" 플래그를 설정하고 1-hop 영향 분석 결과를 확인하는 화면을 구현한다. PRD User Flow Step 7–8을 커버한다.

## 완료 기준

**영향 분석 페이지 (`/impact`)**
- [ ] 전체 요구사항 목록이 표시되고, 각 항목에 "변경 예정" 토글이 있다
  - 토글 → `PATCH /api/requirements/{id}` (`changed: true/false`) 호출
  - `changed=true`인 항목은 시각적으로 구분 (예: 주황색 배지)
- [ ] "영향 분석 실행" 버튼 → `GET /api/impact` 호출
  - 로딩 스피너 표시
- [ ] 분석 결과가 두 섹션으로 표시된다:
  - **영향받음 (Affected):** `affected_items` 목록 — 빨간색 계열 강조
  - **검토 권장 (Review Recommended):** `review_items` 목록 — 노란색 계열 강조
- [ ] 각 결과 항목에 표시:
  - 요구사항 제목
  - 소속 문서명 + `display_label`
  - 연결 근거 문장 (`evidence`)
- [ ] 결과 항목 클릭 시 원문 뷰어 패널 표시:
  - 해당 문서의 `raw_text`를 plain text로 렌더링
  - `char_start`–`char_end` 범위를 하이라이트
  - 해당 위치로 자동 스크롤
- [ ] `changed=true`인 요구사항이 없을 때 "변경 예정으로 표시된 요구사항이 없습니다" 안내 메시지
- [ ] 세션 저장 버튼 → `POST /api/session/save` 호출, 저장 경로 토스트로 표시
- [ ] `npx tsc --noEmit` 오류 없음

## 구현 범위

```
frontend/src/
├── pages/
│   └── ImpactPage.tsx
├── components/
│   ├── RequirementToggleList.tsx   # changed 플래그 토글 목록
│   ├── ImpactResultPanel.tsx       # affected / review 섹션
│   ├── ImpactItem.tsx              # 개별 결과 항목
│   └── DocumentViewer.tsx         # plain text + char offset 하이라이트
├── api/
│   └── impact.ts                   # getImpact, saveSession
└── stores/
    └── impactStore.ts              # impactResult 상태 채움
```

**DocumentViewer 구현:**
- `GET /api/documents/{id}`로 `raw_text` 조회
- `<pre>` 태그로 렌더링
- `char_start`–`char_end` 범위를 `<mark>` 태그로 감싸 하이라이트
- `scrollIntoView()`로 해당 위치 자동 스크롤
