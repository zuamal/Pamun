# F23 — 그래프 노드 디자인 개선

## 개요

`RequirementNode.tsx`의 시각 디자인을 개선한다. 노드 수가 늘어도 가독성을 유지하고, 문서 구분과 허브 노드를 직관적으로 식별할 수 있도록 한다.

---

## 변경 명세

### 레이아웃

```
┌──────────────────────────────┐
│ ● PRD-1                   3↗ │  ← 상단 행: 색상 도트 + ID 라벨 + 연결 수 뱃지
│                              │
│  사용자 인증 기능 구현        │  ← 하단: 제목 (sm, line-clamp-2)
└──────────────────────────────┘
```

### 수치 변경

| 항목 | 현재 | 변경 |
|------|------|------|
| 노드 폭 | 160–220px | 220px 고정 |
| 노드 높이 | 가변 | 80px 고정 |
| ID 라벨 폰트 | 10px | 11–12px (`text-[11px]`) |
| 제목 폰트 | xs (12px) | sm (14px) |
| 제목 줄 수 | line-clamp-3 | line-clamp-2 |

### 문서 구분 인디케이터

- ID 라벨 앞에 6×6px 색상 도트 (원형, `docColors` 팔레트 사용)
- 기존 테두리 색상도 유지 (두 가지 신호 병용)

### 연결 수 뱃지

- 위치: 상단 행 우측
- 값: 해당 노드에 연결된 APPROVED Edge 수 (source + target 모두 집계)
- 형태: `{count}↗` 텍스트, `text-[10px] text-slate-400`
- Edge 수가 0이면 뱃지 숨김

## 완료 기준

### `RequirementNode.tsx` 수정

- [ ] 노드 컨테이너 폭: `w-[220px]` 고정
- [ ] 노드 컨테이너 높이: `h-[80px]` 고정
- [ ] 상단 행 구성:
  - 좌: `●` 색상 도트 (`w-1.5 h-1.5 rounded-full`, `docColors` 팔레트 배경색)
  - 중: 요구사항 ID 라벨 (`text-[11px] font-semibold`)
  - 우: APPROVED 연결 수 뱃지 (`text-[10px] text-slate-400`, 0이면 숨김)
- [ ] 하단 제목: `text-sm line-clamp-2`
- [ ] 기존 impact 상태 스타일(changed 주황 테두리, affected 빨간 링, review_recommended 노란 링)은 그대로 유지
- [ ] 기존 테두리 색상(문서별 구분)은 그대로 유지

### 연결 수 계산

- [ ] `RequirementNode`의 props에 `edgeCount: number`를 추가하거나, 부모(`RequirementGraph.tsx`)에서 노드 데이터 생성 시 해당 노드의 APPROVED edge 수를 계산하여 주입
  - `edges.filter(e => e.status === 'approved' && (e.source_id === req.id || e.target_id === req.id)).length`
