# F17 — Impact Mode on GraphPage

## 개요

GraphPage에 "영향 분석 모드"를 추가한다. 모드 전환 시 노드 클릭이 `changed` 플래그 토글로 동작하고, 변경 즉시 영향받는 노드·엣지가 그래프 위에서 색상으로 시각화된다. 결과 요약은 하단 슬라이드 패널, 원문 확인은 우측 DocumentViewer 패널로 표시한다.

**F11(ImpactPage as separate page)을 대체한다.** ImpactPage는 이 feature 완료 후 Stepper에서 제거되고 `/impact` 라우트는 GraphPage로 리디렉션된다.

**Stepper 변경:** 4단계 → 3단계 (1. 문서 업로드 / 2. 요구사항 검토 / 3. 그래프 & 영향 분석)

---

## 완료 기준

### GraphPage — Impact Mode 토글

- [ ] 그래프 우상단에 "영향 분석 모드" 토글 버튼 추가
  - 기본 모드(엣지 검토): 기존 EdgeReviewPanel, Handle 드래그, NodeDetailPanel 동작
  - 영향 분석 모드: 노드 클릭 → `PATCH /api/requirements/{id}` (`changed` 토글) → 즉시 `GET /api/impact` 호출
  - 두 모드가 상호 배타적으로 동작 (모드 전환 시 선택 상태 초기화)
- [ ] `graphStore`에 `impactMode: boolean` 상태 추가
- [ ] 영향 분석 모드 진입 시 NodeDetailPanel 비활성화 (클릭이 플래그 토글로만 동작)

### 노드 색상 코딩

- [ ] `changed=true` 노드: 주황색 테두리(`ring-2 ring-amber-500`) + 플래그 아이콘(🚩 또는 SVG)
- [ ] `affected` 노드: 빨간 링(`ring-2 ring-red-500`) + 연한 빨간 배경
- [ ] `review_recommended` 노드: 노란 링(`ring-2 ring-yellow-400`) + 연한 노란 배경
- [ ] 영향 분석 모드 OFF 시 모든 색상 코딩 초기화 (노드 원래 스타일로 복원)
- [ ] 영향에 관여한 Edge: 색상 변경(회색 → 관계 유형별 강조색) + strokeWidth 증가

### 하단 슬라이드 패널 (Impact Summary Panel)

- [ ] `GET /api/impact` 응답 수신 시 그래프 하단에서 슬라이드 업 패널 표시
  - 헤더: "영향받음 N개 · 검토 권장 M개"
  - 탭: [영향받음] / [검토 권장]
  - 각 탭에 `ImpactItem` 목록: 요구사항 제목, 문서명, `display_label`, `evidence`
- [ ] `changed` 노드가 없으면 패널 닫힘
- [ ] 패널 높이는 드래그로 조절 가능 (선택적), 기본 높이는 그래프 영역의 1/3

### 우측 DocumentViewer 패널

- [ ] 항목 클릭 시 그래프 우측에서 DocumentViewer 패널 슬라이드 인
  - `GET /api/documents/{document_id}`로 `raw_text` 조회
  - `char_start`–`char_end` 범위 형광펜 하이라이트 + 자동 스크롤
- [ ] 패널 상단에 닫기(×) 버튼
- [ ] 패널이 열린 상태에서 다른 항목 클릭 시 내용 교체 (패널 재오픈 없이)

### Stepper 업데이트

- [ ] `Stepper.tsx`를 3단계로 수정:
  - Step 1: 문서 업로드 (`/`)
  - Step 2: 요구사항 검토 (`/review`)
  - Step 3: 그래프 & 영향 분석 (`/graph`)
- [ ] Step 3 활성화 조건: `edges.filter(APPROVED).length > 0`
- [ ] `/impact` 라우트 → `/graph`로 리디렉션 (기존 F11 라우트 처리)
- [ ] `App.tsx`에서 ImpactPage 라우트 제거 또는 리디렉션으로 교체

### 테스트

- [ ] `npx tsc --noEmit` 오류 없음
- [ ] 시나리오 수동 검증:
  - 영향 분석 모드 ON → 노드 클릭 → 노드 색상 변경 + 하단 패널 슬라이드 업 확인
  - 패널 항목 클릭 → DocumentViewer 패널 열림 + 하이라이트 확인
  - 영향 분석 모드 OFF → 색상 초기화 + 패널 닫힘 확인
  - 모드 전환 중 Handle 드래그(엣지 생성)가 비활성화되는지 확인

## F11과의 관계

F11(ImpactPage)에서 구현한 내용 중:
- `RequirementToggleList`: 제거 (changed 토글이 그래프 노드 클릭으로 대체)
- `ImpactResultPanel` + `ImpactItem`: `ImpactSummaryPanel`로 이동하여 재사용
- `DocumentViewer`: 우측 패널로 재배치하여 재사용
- `impactStore`: `graphStore`로 통합 (`impactResult` 상태를 graphStore로 이동)

F11이 완료된 상태라면 컴포넌트를 재사용하고 위치와 트리거만 변경한다.
