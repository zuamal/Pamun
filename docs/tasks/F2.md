# F2 — Document Upload & Text Extraction

## 개요

파일 업로드 API와 포맷별 plain text 추출 서비스를 구현한다. 추출된 텍스트가 이후 LLM 파싱(F3)의 입력이 된다.

## 완료 기준

- [ ] `POST /api/documents/upload` — multipart/form-data로 파일 최대 5개 업로드
  - 지원 포맷: `.md`, `.docx`, `.pdf`
  - 지원 포맷 외 파일 업로드 시 422 반환
  - 6개 이상 업로드 시 422 반환
  - 응답: 업로드된 `Document` 목록
- [ ] `GET /api/documents` — 업로드된 문서 목록 반환
- [ ] `GET /api/documents/{id}` — 문서 상세 (`raw_text` 포함) 반환. 없으면 404
- [ ] `DELETE /api/documents/{id}` — 문서 삭제. 없으면 404
- [ ] 포맷별 텍스트 추출이 정상 동작한다:
  - Markdown: 원문 그대로 (파싱 없이 raw text)
  - DOCX: `python-docx`로 단락 텍스트 추출, 줄바꿈으로 연결
  - PDF: `pdfplumber`로 페이지별 텍스트 추출, 줄바꿈으로 연결
  - Scanned PDF(텍스트 레이어 없음): 422 반환, 메시지: `"텍스트를 추출할 수 없습니다. 텍스트 기반 PDF만 지원합니다."`
- [ ] `cd backend && mypy app/ --strict` 오류 없음
- [ ] `pytest tests/test_documents.py -v` 통과:
  - Markdown 업로드 → `raw_text` 검증
  - DOCX 업로드 → `raw_text` 검증
  - PDF 업로드 → `raw_text` 검증
  - 포맷 오류, 개수 초과, 없는 ID 조회 케이스

## 구현 범위

```
backend/
├── app/
│   ├── api/
│   │   ├── __init__.py
│   │   └── documents.py      # 4개 엔드포인트
│   └── services/
│       ├── __init__.py
│       └── document_service.py  # 텍스트 추출 로직 (포맷별 분기)
├── tests/
│   ├── test_documents.py
│   └── fixtures/             # 테스트용 샘플 파일 (small_sample.md, small_sample.docx, small_sample.pdf)
└── requirements.txt          # python-docx, pdfplumber 추가
```

**Document ID 생성:** `uuid4()` 사용.
