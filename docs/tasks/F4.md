# F4 — Requirement Management

## 개요

PM이 파싱 결과를 검토하며 요구사항을 수정·분리·병합·삭제할 수 있는 API를 구현한다.

## 완료 기준

- [ ] `PATCH /api/requirements/{id}` — 제목(`title`) 또는 변경 플래그(`changed`) 수정
  - 없는 ID 요청 시 404
  - 두 필드 모두 null이면 422
  - 수정된 `Requirement` 반환
- [ ] `DELETE /api/requirements/{id}` — 요구사항 삭제
  - 해당 요구사항과 연결된 모든 Edge도 함께 삭제
  - 없는 ID 요청 시 404
- [ ] `POST /api/requirements/merge` — 요구사항 병합
  - 같은 문서(`document_id`)에 속한 요구사항만 병합 가능. 아니면 422
  - 인접한 span끼리만 병합 가능 (`char_end` of one == `char_start` of next, 또는 whitespace만 사이에 있음). 아니면 422
  - 병합 결과: `char_start` = 최솟값, `char_end` = 최댓값, `original_text` = 연결, `title`은 첫 번째 항목의 title 유지
  - 기존 요구사항들을 삭제하고 새 `Requirement`(새 ID)를 생성
  - 기존 요구사항들에 연결된 Edge는 새 ID로 재연결
- [ ] `POST /api/requirements/split` — 요구사항 분리
  - `split_offset`이 `original_text` 범위를 벗어나면 422
  - `split_offset`이 0 또는 `len(original_text)`이면 422 (빈 파트 생성 불가)
  - 분리 결과: 두 개의 새 `Requirement`(새 ID 각각)
  - 기존 요구사항에 연결된 Edge는 첫 번째 파트에 재연결
- [ ] `cd backend && mypy app/ --strict` 오류 없음
- [ ] `pytest tests/test_requirements.py -v` 통과:
  - PATCH: title 수정, changed 플래그 토글
  - DELETE: 삭제 + 연결된 Edge 삭제 확인
  - Merge: 정상 케이스, 다른 문서 오류, 비인접 오류
  - Split: 정상 케이스, offset 범위 오류

## 구현 범위

```
backend/
├── app/
│   ├── api/
│   │   └── requirements.py   # PATCH, DELETE, POST merge, POST split
│   └── services/
│       └── requirement_service.py  # merge/split 비즈니스 로직
├── tests/
│   └── test_requirements.py
```
