# F13 — Global Architecture: Tailwind + App Shell + Toast + Skeleton

## 개요

프론트엔드 전체의 기반 인프라를 교체한다. Tailwind CSS 도입, 좌측 Sidebar + Stepper App Shell 구성, sonner Toast 시스템, Skeleton/Empty State 패턴 정의. F14(페이지별 UX)가 이 feature를 전제로 한다.

---

## 완료 기준

### Tailwind CSS 설정

- [ ] `tailwind.config.js`와 `postcss.config.js`가 추가되고 `vite.config.ts`에 통합된다
- [ ] `src/index.css`에 `@tailwind base/components/utilities` 지시어가 추가된다
- [ ] 기존 코드에 남아 있는 인라인 `style={{...}}` prop이 전부 제거되고 Tailwind 클래스로 대체된다
- [ ] `npx tsc --noEmit` 오류 없음

### Store Hydration (새로고침 복원)

- [ ] `src/hooks/useHydrate.ts`가 구현된다:
  - 앱 최초 마운트 시 `GET /api/documents`, `GET /api/requirements`, `GET /api/edges` 를 `Promise.all`로 병렬 호출
  - 결과를 `documentStore`, `graphStore`에 각각 저장
  - `hydrating: boolean` 상태를 반환
- [ ] `App.tsx`에서 `useHydrate()`를 호출하고, `hydrating === true`인 동안 전체 화면에 로딩 스피너를 표시하여 라우트 렌더링을 차단한다
- [ ] Hydration 완료 후 store가 채워진 상태에서 ProtectedRoute가 조건을 평가한다
- [ ] 백엔드 서버가 실행 중이고 기존 세션이 인메모리에 남아 있는 경우, 새로고침 후에도 기존 경로가 유지된다

### App Shell (Sidebar + Stepper)

- [ ] `src/components/AppShell.tsx`가 구현된다:
  - 좌측에 `w-56` 고정 Sidebar, 우측에 `flex-1` 콘텐츠 영역
  - Sidebar 안에 `Stepper` 컴포넌트와 프로젝트 이름(Pamun)이 렌더링된다
- [ ] `src/components/Stepper.tsx`가 구현된다:
  - 4단계: 문서 업로드 / 요구사항 검토 / 의존관계 그래프 / 영향 분석
  - 현재 경로(`useLocation`)에 따라 활성 단계가 강조된다
  - 각 단계의 활성화 조건을 Zustand store에서 읽어 비활성 단계는 시각적으로 dim 처리
  - 비활성 단계 클릭 시 조건 미충족 안내 툴팁 표시 (navigate 차단)
  - 조건 미충족 경로 직접 접근 시 Hydration 완료 후 재평가하여 미충족이면 `/`로 리디렉션
- [ ] Stepper 활성화 조건이 store에서 파생된다:
  - Step 2·3: `graphStore.requirements.length > 0`
  - Step 4: `graphStore.edges`에 `APPROVED` 상태 Edge가 1개 이상
- [ ] `App.tsx`가 모든 라우트를 `AppShell`로 감싼다

### Toast (sonner)

- [ ] `sonner` 패키지가 설치된다
- [ ] `main.tsx`에 `<Toaster position="bottom-right" />` 추가
- [ ] `src/lib/toast.ts`에 프로젝트 공통 toast 헬퍼 함수 정의:
  ```ts
  export const toastSuccess = (msg: string) => toast.success(msg)
  export const toastError = (msg: string) => toast.error(msg)
  ```
- [ ] 기존 fetch 함수의 오류 처리가 `toastError()`를 호출하도록 정리된다 (각 페이지에서 개별 catch 불필요)

### Skeleton UI & Empty State

- [ ] `src/components/Skeleton.tsx`: 단일 Skeleton 블록 컴포넌트 (width/height Tailwind 클래스를 prop으로 받음)
- [ ] `src/components/EmptyState.tsx`: 제목, 설명, 선택적 액션 버튼을 받는 공통 Empty State 컴포넌트
- [ ] `src/components/RequirementListSkeleton.tsx`: RequirementList 로딩 시 사용할 Skeleton 구성 (3–5개 항목 형태)
- [ ] `src/components/DocumentListSkeleton.tsx`: DocumentList 로딩 시 사용할 Skeleton 구성

## 구현 범위

```
frontend/
├── tailwind.config.js              # 신규
├── postcss.config.js               # 신규
├── src/
│   ├── index.css                   # @tailwind 지시어 추가
│   ├── App.tsx                     # useHydrate 호출, AppShell로 감싸도록 수정
│   ├── main.tsx                    # <Toaster /> 추가
│   ├── hooks/
│   │   └── useHydrate.ts           # 신규: 앱 마운트 시 백엔드 상태 복원
│   ├── lib/
│   │   └── toast.ts                # 신규: toast 헬퍼
│   └── components/
│       ├── AppShell.tsx            # 신규
│       ├── Stepper.tsx             # 신규
│       ├── Skeleton.tsx            # 신규
│       ├── EmptyState.tsx          # 신규
│       ├── RequirementListSkeleton.tsx  # 신규
│       └── DocumentListSkeleton.tsx     # 신규
```

기존 컴포넌트의 인라인 `style` 제거 및 Tailwind 클래스 적용은 이 feature 내에서 함께 진행한다.
