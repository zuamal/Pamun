# F9 — Frontend: Document Upload & Requirement Review

## 개요

문서 업로드부터 요구사항 파싱, PM 검토(수정/분리/병합/삭제)까지 PRD User Flow Step 1–3을 커버하는 프론트엔드 화면을 구현한다.

## 완료 기준

**문서 업로드 페이지 (`/`)**
- [ ] 파일 선택 또는 드래그&드롭으로 최대 5개 파일 업로드 가능
- [ ] 지원 포맷(`.md`, `.docx`, `.pdf`) 외 파일 선택 시 UI 레벨에서 오류 표시
- [ ] 업로드 진행 중 로딩 상태 표시
- [ ] 업로드 성공 시 문서 목록 표시 (파일명, 포맷)
- [ ] 문서 삭제 버튼 → DELETE 호출 → 목록에서 제거
- [ ] "파싱 시작" 버튼 → `POST /api/parse` 호출
  - 로딩 스피너 표시 (최대 120초 대기)
  - 성공 시 요구사항 검토 페이지(`/review`)로 이동

**요구사항 검토 페이지 (`/review`)**
- [ ] 파싱된 요구사항 목록 표시: ID, 제목, 원문 일부(앞 100자), 소속 문서명
- [ ] 제목 인라인 편집 → `PATCH /api/requirements/{id}` 호출
- [ ] 요구사항 삭제 버튼 → `DELETE /api/requirements/{id}` 호출 → 목록에서 제거
- [ ] 요구사항 분리(Split) UI:
  - 클릭 시 원문 텍스트가 표시되고 분리 위치(offset)를 지정할 수 있다
  - 확인 → `POST /api/requirements/split` 호출
- [ ] 요구사항 병합(Merge) UI:
  - 체크박스로 2개 선택 → "병합" 버튼 활성화
  - 확인 → `POST /api/requirements/merge` 호출
  - 다른 문서 간 병합 시도 시 422 에러 메시지 표시
- [ ] "의존관계 추론 시작" 버튼 → `POST /api/edges/infer` 호출 → 그래프 페이지(`/graph`)로 이동
- [ ] API 오류 시 토스트 메시지로 에러 내용 표시

## 구현 범위

```
frontend/src/
├── pages/
│   ├── UploadPage.tsx
│   └── ReviewPage.tsx
├── components/
│   ├── FileDropzone.tsx       # 드래그&드롭 + 파일 선택
│   ├── DocumentList.tsx
│   ├── RequirementList.tsx
│   ├── RequirementItem.tsx    # 인라인 편집, 분리, 선택 체크박스
│   └── SplitModal.tsx         # 분리 offset 지정 UI
├── api/
│   ├── documents.ts           # uploadDocuments, listDocuments, deleteDocument
│   └── requirements.ts        # listRequirements, updateRequirement, mergeRequirements, splitRequirement, deleteRequirement, inferEdges
└── stores/
    ├── documentStore.ts       # documents 상태 채움
    └── graphStore.ts          # requirements 상태 채움
```
