# F10 — Frontend: Graph Visualization & Edge Review

## 개요

React Flow로 요구사항 의존관계 그래프를 시각화하고, PM이 LLM이 추론한 Edge를 승인/거부/수정하거나 수동으로 Edge를 추가할 수 있는 화면을 구현한다. PRD User Flow Step 5–6을 커버한다.

## 완료 기준

**그래프 페이지 (`/graph`)**
- [ ] 승인된(APPROVED) Edge만으로 구성된 의존관계 그래프가 React Flow로 렌더링된다
- [ ] 노드: 요구사항 ID + 제목 표시. 문서별로 색상 또는 테두리가 구분된다 (document별 클러스터링)
- [ ] 엣지: `depends_on`은 화살표(방향 있음), `related_to`는 양방향 화살표로 구분
- [ ] 노드 클릭 시 우측 패널에 상세 표시:
  - 원문(`original_text`) 전체
  - 소속 문서명 + `display_label`
  - 연결된 Edge 목록 (상대 요구사항 제목, relation_type, evidence)
- [ ] 줌·패닝 인터랙션 동작
- [ ] "PENDING Edge 검토" 패널:
  - PENDING 상태 Edge 목록 표시 (source 제목 → target 제목, evidence, confidence)
  - 각 Edge에 승인/거부 버튼 → `PATCH /api/edges/{id}` 호출
  - 승인 시 그래프에 즉시 추가됨
- [ ] 수동 Edge 추가:
  - 두 노드를 선택 후 "연결 추가" → `POST /api/edges` 호출
  - relation_type 선택 (depends_on / related_to), evidence 입력
- [ ] Edge 삭제: 그래프 또는 목록에서 선택 → `DELETE /api/edges/{id}` 호출
- [ ] "영향 분석으로" 버튼 → `/impact` 페이지로 이동
- [ ] `npx tsc --noEmit` 오류 없음

## 구현 범위

```
frontend/src/
├── pages/
│   └── GraphPage.tsx
├── components/
│   ├── RequirementGraph.tsx   # React Flow 래퍼
│   ├── RequirementNode.tsx    # 커스텀 노드 컴포넌트
│   ├── EdgeReviewPanel.tsx    # PENDING Edge 검토 목록
│   ├── NodeDetailPanel.tsx    # 노드 클릭 시 상세 패널
│   └── AddEdgeModal.tsx       # 수동 Edge 추가 모달
├── api/
│   └── edges.ts               # listEdges, createEdge, updateEdge, deleteEdge
└── stores/
    └── graphStore.ts           # edges, 줌/레이아웃 상태 채움
```

**React Flow 레이아웃:** 초기 레이아웃은 `dagre` 라이브러리로 자동 배치 (top-down DAG). 이후 드래그로 수동 조정 가능.

**상태 ownership:** 줌 레벨, 노드 위치는 `graphStore`에서 관리. 서버와 동기화하지 않음 (ADR-4).
