# F20 — GraphPage / ImpactPage UI/UX 재구성

## 개요

GraphPage에 통합되어 있던 영향 분석 기능을 별도 ImpactPage(Step 4)로 분리하고, 각 페이지의 레이아웃을 역할에 맞게 재구성한다.

| 페이지 | 역할 | 변경 내용 |
|--------|------|-----------|
| GraphPage (Step 3) | Edge 관리 전용 | Impact 모드 토글 제거, 전체 높이 그래프, Edge hover 강조 |
| ImpactPage (Step 4) | 영향 분석 전용 | 리사이저블 좌측 패널 + 우측 그래프/DocumentViewer 탭 |
| Stepper | 4단계 복원 | Step 4 "영향 분석" 추가 |

---

## 완료 기준

### GraphPage 리팩토링

- [ ] Impact 모드 토글(`ImpactModeToggle`) 컴포넌트 및 관련 로직 제거
- [ ] `ImpactSummaryPanel`(하단 슬라이드업 패널) 제거. 그래프가 전체 높이를 사용한다
- [ ] DocumentViewer 슬라이드인 패널 GraphPage에서 제거 (ImpactPage로 이동)
- [ ] 우측 패널: Edge 검토 카드 리스트(상단) + 노드 상세(하단) 구성 유지
- [ ] Edge 검토 카드 hover 시 그래프의 해당 Edge 강조:
  - 카드 hover → 해당 edge `strokeWidth` 증가 + 강조 색상 적용
  - 카드 hover out → 원래 스타일 복원
  - React Flow `useReactFlow().setEdges` 또는 edge `className` 동적 업데이트로 구현
- [ ] `graphStore`에서 `impactResult`, `isImpactMode` 상태 제거 (ImpactPage 전용으로 이동)

### ImpactPage 재구현

- [ ] 레이아웃 구성:
  ```
  ┌──────────┬─────────────────────────────────┐
  │ 좌측패널 │ [그래프 뷰] [DocumentViewer]    │
  │(리사이저)│                                 │
  │ 요구사항 │  (탭 전환 영역)                 │
  │ + changed│                                 │
  │   토글   │                                 │
  ├──────────┤                                 │
  │ 영향결과 │                                 │
  └──────────┴─────────────────────────────────┘
  ```
- [ ] 좌측 패널:
  - 리사이저블 (`react-resizable-panels` 또는 직접 구현). 최소 폭 200px, 최대 폭 480px, 기본 폭 280px
  - 상단: 요구사항 카드 리스트. 각 카드에 changed 토글 포함. 토글 활성화 시 카드 주황 강조 즉시 반영
  - 하단: 영향 분석 결과 — affected N개 / review_recommended M개 + 항목 목록(요구사항 제목, 문서명, display_label)
  - changed 토글 변경 시 즉시 `GET /api/impact` 호출하여 결과 갱신
- [ ] 우측 영역 탭:
  - 기본 탭: 그래프 뷰 — changed(주황 테두리) / affected(빨간 링) / review_recommended(노란 링) 노드 하이라이트. 영향 관련 Edge 색상 강조
  - 두 번째 탭: DocumentViewer — 선택된 영향 항목의 원문 범위 하이라이트
  - 영향 결과 항목 클릭 시 DocumentViewer 탭으로 자동 전환
- [ ] 세션 저장 버튼을 ImpactPage 헤더에도 배치 (GraphPage와 동일 위치)
- [ ] `GET /api/impact` 응답을 ImpactPage 전용 로컬 상태로 관리 (graphStore 분리)

### Stepper 4단계 복원

- [ ] `Stepper.tsx` 단계 배열 업데이트:
  ```ts
  [
    { label: '문서 업로드', path: '/' },
    { label: '요구사항 검토', path: '/review' },
    { label: '그래프 & Edge 검토', path: '/graph' },
    { label: '영향 분석', path: '/impact' },
  ]
  ```
- [ ] Step 4(`/impact`) 활성화 조건: `edges.filter(e => e.status === 'APPROVED').length > 0`
- [ ] `ProtectedRoute`에서 `/impact` 경로에 동일 조건 적용
- [ ] `ImpactPage`로의 리디렉션 제거 (현재 ImpactPage가 GraphPage로 리디렉션하는 코드 제거)
- [ ] `useHydrate` 훅은 그대로 유지 (새로고침 시 store hydration 동작 동일)

### 데모 모드 호환

- [ ] `demoApi.ts`의 mock `getImpact()` 함수가 ImpactPage에서도 정상 동작 확인
- [ ] `VITE_DEMO_MODE=true` 빌드에서 ImpactPage의 changed 토글 → `impactCompute.ts` 클라이언트 계산으로 분기

### 타입 체크

- [ ] `npx tsc --noEmit` 오류 없음
