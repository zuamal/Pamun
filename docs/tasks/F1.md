# F1 — Backend Foundation

## 개요

FastAPI 앱 골격, 의존성, 스토리지 레이어를 구축한다. 이후 모든 backend feature(F2–F7)의 기반이 된다.

## 완료 기준

- [ ] `uvicorn app.main:app --reload --port 8000` 실행 시 서버가 정상 기동된다
- [ ] `GET /health` → `{"status": "ok"}` 응답
- [ ] `GET /docs` (Swagger UI) 접근 가능
- [ ] `backend/app/models/` 아래 5개 도메인 모델 파일이 존재하고 ADR Appendix A 스키마와 일치한다:
  - `document.py` (`Document`, `DocumentFormat`)
  - `requirement.py` (`Requirement`, `RequirementLocation`)
  - `edge.py` (`Edge`, `RelationType`, `EdgeStatus`)
  - `impact.py` (`ImpactItem`, `ImpactResult`, `ImpactLevel`)
  - `api.py` (모든 Request/Response 모델)
- [ ] `backend/app/storage/store.py`에 인메모리 스토어가 구현된다:
  - `documents: dict[str, Document]`
  - `requirements: dict[str, Requirement]`
  - `graph: nx.DiGraph` (networkx)
  - `edges: dict[str, Edge]`
  - 앱 전역에서 singleton으로 접근 가능
- [ ] `cd backend && mypy app/ --strict` 오류 없음
- [ ] `cd backend && pytest tests/ -v` — 스토어 unit test 통과 (CRUD 기본 동작 확인)

## 구현 범위

```
backend/
├── app/
│   ├── main.py               # FastAPI 앱 생성, 라우터 등록, CORS 설정
│   ├── models/
│   │   ├── __init__.py
│   │   ├── document.py
│   │   ├── requirement.py
│   │   ├── edge.py
│   │   ├── impact.py
│   │   └── api.py
│   └── storage/
│       ├── __init__.py
│       └── store.py          # AppStore 클래스 (singleton)
├── tests/
│   ├── __init__.py
│   └── test_store.py
├── requirements.txt          # fastapi, uvicorn, pydantic, networkx, mypy, pytest
└── pyproject.toml            # mypy 설정 (strict)
```

**CORS:** 개발 환경에서 `http://localhost:5173` (Vite 기본 포트) 허용.
