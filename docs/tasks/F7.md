# F7 — Session Persistence

## 개요

현재 앱 상태(문서, 요구사항, 그래프, Edge)를 JSON 파일로 저장하고 복원하는 세션 API를 구현한다.

## 완료 기준

- [ ] `POST /api/session/save` — 현재 인메모리 상태를 JSON 파일로 저장
  - 저장 경로: `{project_root}/sessions/{timestamp}.session.json`
  - 응답: `{"filepath": "...", "saved_at": "ISO8601"}`
  - 저장 데이터: `documents`, `requirements`, `edges` 전체 (networkx 그래프는 edges로 재구성 가능하므로 저장 불필요)
- [ ] `POST /api/session/load` — JSON 파일에서 상태 복원
  - 요청 바디: `{"filepath": "..."}`
  - 파일이 존재하지 않으면 404
  - 파일 포맷이 유효하지 않으면 422
  - 복원 성공 시 현재 상태를 완전히 덮어쓴다
  - networkx 그래프는 복원된 edges 중 `APPROVED` Edge로 재구성
- [ ] `POST /api/session/reset` — 전체 인메모리 상태 초기화
  - documents, requirements, edges, graph 모두 비움
  - 응답: `{"status": "reset"}`
- [ ] save → reset → load 사이클 후 상태가 동일하다 (round-trip 보장)
- [ ] `cd backend && mypy app/ --strict` 오류 없음
- [ ] `pytest tests/test_session.py -v` 통과:
  - save → load round-trip (데이터 일치 검증)
  - reset 후 빈 상태 확인
  - 없는 파일 load 404
  - load 후 APPROVED edge가 그래프에 반영됨

## 구현 범위

```
backend/
├── app/
│   ├── api/
│   │   └── session.py        # 3개 엔드포인트
│   └── services/
│       └── session_service.py  # 직렬화/역직렬화 로직
├── tests/
│   └── test_session.py
```

**직렬화:** Pydantic의 `model_dump()` / `model_validate()` 사용. `datetime` 필드는 ISO8601 문자열로 직렬화.

**sessions/ 디렉토리:** 앱 기동 시 없으면 자동 생성. `.gitignore`에 `*.session.json` 이미 포함됨.
