# F14 — Page-level UX: 4개 페이지 고도화

## 개요

4개 페이지(UploadPage, ReviewPage, GraphPage, ImpactPage)의 UI/UX를 고도화한다. F13(Tailwind + App Shell)이 완료된 이후 진행한다.

---

## 완료 기준

### UploadPage

- [ ] **드롭존 시각화:** 파일 드래그 진입(`dragover`) 시 테두리 색상 변경 + 아이콘 scale-up 애니메이션 (`transition`, `scale-110`). 드래그 이탈 시 원상 복귀
- [ ] **파일 카드:** 업로드된 파일을 각각 카드 형태로 표시:
  - 포맷별 아이콘 (`.md` / `.docx` / `.pdf` 구분)
  - 파일명과 용량 (`KB` / `MB` 단위)
  - 삭제 버튼 (휴지통 아이콘, hover 시 빨간색으로 변경)
- [ ] 파일이 없을 때 `EmptyState` 컴포넌트 표시 ("문서를 업로드해주세요" + 파일 선택 유도 버튼)
- [ ] `npx tsc --noEmit` 오류 없음

### ReviewPage

- [ ] **Accordion 그룹화:** 요구사항 목록을 `document_id` 기준으로 그룹화하여 Accordion으로 표시
  - 각 Accordion 헤더: 문서명 + 요구사항 수 뱃지
  - 기본 상태: 모두 펼쳐짐. 헤더 클릭으로 토글
- [ ] **다중 선택 + Floating Action Bar:**
  - 각 요구사항 항목에 체크박스 추가
  - 1개 이상 선택 시 화면 하단 중앙에 Floating Action Bar 슬라이드 업
  - Action Bar: 선택된 항목 수 표시 + "병합" 버튼(2개 이상 선택 시 활성) + "삭제" 버튼 + "선택 해제" 버튼
  - 병합/삭제 성공 시 Toast + 목록 갱신
- [ ] **Split 모달 개선:** SplitModal 내 offset 입력 필드의 레이아웃과 스타일만 정돈 (로직 변경 없음). 원문 텍스트를 표시하고 offset 위치를 커서로 미리보기
- [ ] 요구사항이 없을 때 `EmptyState` 표시 ("파싱을 먼저 실행해주세요" + Upload 페이지 이동 버튼)
- [ ] `npx tsc --noEmit` 오류 없음

### GraphPage

- [ ] **Handle 드래그 Edge 생성:**
  - 기존 "노드 선택 → 연결 추가 버튼" 방식을 제거하고 `AddEdgeModal`에서 연결된 수동 트리거 버튼도 제거
  - 각 `RequirementNode`에 React Flow `<Handle>` 추가 (source: 우측, target: 좌측)
  - 핸들에서 드래그 → 다른 노드의 핸들에 드롭 시 `AddEdgeModal`이 열리고 relation_type·evidence 입력 후 `POST /api/edges` 호출
  - 연결 중 `ConnectionLine` 컴포넌트로 드래그 경로 시각화
  - Zustand `graphStore`에서 임시 연결 중인 source 노드 상태 관리
- [ ] **미니맵:** `<MiniMap />` 컴포넌트를 우측 하단에 추가 (React Flow 내장)
- [ ] **필터 컨트롤:** 그래프 상단 또는 Sidebar 하단에 필터 UI 추가:
  - 문서별 필터: 체크박스로 특정 문서의 노드만 표시
  - Edge 상태별 필터: PENDING / APPROVED 토글 (REJECTED는 항상 숨김)
  - 필터 상태는 `graphStore`에서 관리하고 노드/엣지 렌더링 시 적용
- [ ] **Edge 검토 패널 UX:**
  - 각 PENDING Edge 항목에 신뢰도(`confidence`) 점수를 진행률 바 또는 색상으로 시각화
  - 승인/거부 버튼을 더 크게 배치하고 키보드 단축키 힌트 표시 (구현은 선택)
  - 승인·거부 즉시 목록에서 항목 제거 (낙관적 UI)
- [ ] `npx tsc --noEmit` 오류 없음

### ImpactPage

- [ ] **커스텀 Toggle:**
  - '변경 예정' 토글을 `<input type="checkbox">` 기반 Tailwind 커스텀 스위치로 교체
  - `checked` 상태: 주황색 배경, 흰색 thumb. `unchecked`: 회색
  - 토글 transition 애니메이션 (`transition-all duration-200`)
- [ ] **Pulse 애니메이션:**
  - `changed=true`인 요구사항이 1개 이상 존재할 때 "영향 분석 실행" 버튼에 `animate-pulse` 클래스 적용
  - 영향 분석 실행 중(로딩)에는 pulse 제거하고 로딩 스피너로 교체
- [ ] **DocumentViewer 하이라이트 강화:**
  - `char_start`–`char_end` 범위를 `<mark>` 태그 대신 `<span class="bg-yellow-200 font-semibold px-0.5 rounded">` 형태로 교체하여 형광펜 효과 적용
  - 하이라이트 범위로 자동 스크롤 유지
- [ ] 요구사항이나 Edge가 없을 때 적절한 `EmptyState` 표시
- [ ] `npx tsc --noEmit` 오류 없음

## 구현 범위

```
frontend/src/
├── pages/
│   ├── UploadPage.tsx              # 드롭존 애니메이션, 파일 카드
│   ├── ReviewPage.tsx              # Accordion, Floating Action Bar
│   ├── GraphPage.tsx               # 필터 컨트롤, 미니맵
│   └── ImpactPage.tsx              # 커스텀 Toggle, Pulse, 하이라이트
├── components/
│   ├── FileDropzone.tsx            # dragover 상태 애니메이션
│   ├── DocumentList.tsx            # 파일 카드 (아이콘, 용량, 삭제)
│   ├── RequirementList.tsx         # Accordion 래퍼
│   ├── RequirementItem.tsx         # 체크박스 추가
│   ├── FloatingActionBar.tsx       # 신규: 다중 선택 액션 바
│   ├── SplitModal.tsx              # 레이아웃/스타일 개선 (로직 유지)
│   ├── RequirementNode.tsx         # Handle 추가
│   ├── RequirementGraph.tsx        # MiniMap, ConnectionLine, 필터 적용
│   ├── EdgeReviewPanel.tsx         # confidence 시각화, 버튼 크기 개선
│   ├── AddEdgeModal.tsx            # 수동 버튼 제거 (Handle 드래그로만 열림)
│   ├── GraphFilter.tsx             # 신규: 문서별·상태별 필터
│   ├── DocumentViewer.tsx          # 형광펜 하이라이트
│   └── RequirementToggleList.tsx   # 커스텀 Toggle, Pulse 버튼
└── stores/
    └── graphStore.ts               # 필터 상태, 임시 연결 소스 노드 추가
```
