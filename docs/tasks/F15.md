# F15 — Sample Bundle Loader

## 개요

`docs/dummy/`의 샘플 문서 번들을 데모·QA 목적으로 빠르게 적재하는 기능을 구현한다.

- **Backend:** 번들 목록 조회 + 번들 적재 API, pytest fixture
- **Frontend:** UploadPage의 "샘플 불러오기" UI

번들 3개:

| 번들 | 파일 |
|------|------|
| BookFlow | PRD.md, 기능명세서.docx, QA_체크리스트.pdf |
| LearnHub | PRD.md, API_명세서.docx, 데이터모델.md, 알림명세서.md, 테스트계획서.pdf |
| MediBook | PRD.md, API_명세서.docx, 개인정보처리방침.pdf, 비기능_요구사항.md |

---

## 완료 기준

### Backend

- [ ] `GET /api/dummy/bundles` — 사용 가능한 번들 목록 반환
  ```json
  [
    {"name": "BookFlow", "files": ["PRD.md", "기능명세서.docx", "QA_체크리스트.pdf"]},
    {"name": "LearnHub", "files": [...]},
    {"name": "MediBook",  "files": [...]}
  ]
  ```
- [ ] `POST /api/dummy/load/{bundle_name}` — 번들 적재
  - 존재하지 않는 `bundle_name` 요청 시 404
  - 적재 전 현재 인메모리 상태를 초기화 (session reset과 동일)
  - `docs/dummy/{bundle_name}/` 경로의 파일을 읽어 기존 `document_service`의 텍스트 추출 로직 재사용
  - 적재된 `Document` 목록 반환
- [ ] 더미 경로는 `Path(__file__).parents[3] / "docs" / "dummy"` 로 resolve (하드코딩 금지)
- [ ] `cd backend && mypy app/ --strict` 오류 없음

### pytest fixtures

- [ ] `backend/tests/conftest.py`에 번들 픽스처 추가:
  ```python
  @pytest.fixture
  def bookflow_bundle():
      """BookFlow 번들의 실제 파일 경로 목록 반환"""
      ...

  @pytest.fixture
  def learnhub_bundle(): ...

  @pytest.fixture
  def medibook_bundle(): ...
  ```
  - 각 fixture는 `Path` 객체 목록을 반환하고 실제 파일 존재 여부를 검증한다
- [ ] `backend/tests/test_dummy.py` 작성:
  - `GET /api/dummy/bundles` 응답에 3개 번들이 포함됨
  - `POST /api/dummy/load/BookFlow` 성공 + 반환된 Document 수가 3개
  - 번들 적재 후 `GET /api/documents` 응답이 동일 결과
  - 존재하지 않는 번들 이름 → 404
- [ ] 기존 통합 테스트(`test_parse.py`, `test_edges.py` 등)에서 번들 fixture를 활용하여 실제 파일 기반 E2E 시나리오 추가 (선택적)

### Frontend

- [ ] UploadPage에 "샘플 불러오기" 버튼 추가 (드롭존 하단 또는 파일 목록 영역)
- [ ] 버튼 클릭 시 번들 선택 드롭다운/모달 표시:
  - 번들 이름과 포함 파일 목록 표시 (`GET /api/dummy/bundles` 결과)
- [ ] 번들 선택 시:
  - 확인 다이얼로그: "현재 작업이 초기화됩니다. 계속하시겠습니까?"
  - 확인 → `POST /api/dummy/load/{bundle_name}` 호출
  - 성공 시 documentStore 갱신 + Toast("BookFlow 번들을 불러왔습니다")
  - 실패 시 `toastError()` 호출
- [ ] 적재 중 버튼 로딩 상태 표시
- [ ] `npx tsc --noEmit` 오류 없음

## 구현 범위

```
backend/
├── app/
│   ├── api/
│   │   └── dummy.py              # 신규: GET /api/dummy/bundles, POST /api/dummy/load/{name}
│   └── services/
│       └── dummy_service.py      # 신규: 번들 경로 resolve, 파일 로드 오케스트레이션
├── tests/
│   ├── conftest.py               # 번들 fixture 추가
│   └── test_dummy.py             # 신규

frontend/src/
├── api/
│   └── dummy.ts                  # 신규: listBundles(), loadBundle(name)
└── pages/
    └── UploadPage.tsx            # "샘플 불러오기" 버튼 + 번들 선택 UI 추가
```

**라우터 등록:** `app/main.py`에 `dummy` 라우터 추가.

**DUMMY_ROOT 경로 계산:**
```python
# backend/app/services/dummy_service.py
from pathlib import Path

DUMMY_ROOT = Path(__file__).parents[3] / "docs" / "dummy"
```
