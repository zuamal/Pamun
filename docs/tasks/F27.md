# F27 — 글로벌 UI 비주얼 고도화

## 개요

폰트·색상 체계·마이크로 인터랙션·다크 모드를 통일된 디자인 시스템으로 정립한다. 5가지 개선을 단일 Feature로 묶는다: Pretendard 폰트 + 필 배지, Glassmorphism 패널, 시스템 다크 모드, 마이크로 애니메이션(DOM + Canvas), Skeleton shimmer + Framer Motion 트랜지션.

---

## 완료 기준

### Step 1 — Pretendard 폰트 + 필 배지 (FR-13.1)

- [ ] `index.html` `<head>`에 Pretendard CDN 링크 추가:
  ```html
  <link rel="stylesheet"
    href="https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable-dynamic-subset.min.css" />
  ```
- [ ] `tailwind.config.js`의 `fontFamily.sans`에 `'Pretendard Variable'` 지정:
  ```js
  fontFamily: { sans: ['Pretendard Variable', 'Pretendard', 'system-ui', 'sans-serif'] }
  ```
- [ ] 전체 앱에 `font-sans` 클래스 적용 확인 (`body` 또는 AppShell 루트)
- [ ] 문서 고유 색상 배지를 `rounded-full` 필 배지로 교체:
  - 기존: 원형 `w-3 h-3` 컬러 도트
  - 변경: `<span class="rounded-full px-2 py-0.5 text-xs font-medium" style={{backgroundColor: color}}>` + 파일명 (앞 8자 + 말줄임)
  - 적용 위치: Sidebar 문서 목록, ReviewPage 그룹 헤더, GraphLegend, Edge 카드 문서명 표시

### Step 2 — Glassmorphism 패널 (FR-13.2)

- [ ] `tailwind.config.js`에 `backdropBlur` 활성화 확인 (기본 포함)
- [ ] 공통 유틸 클래스 정의 (CSS 또는 tailwind `@layer components`):
  ```css
  .glass-panel {
    @apply bg-white/70 backdrop-blur-md border border-white/30 shadow-lg;
  }
  ```
- [ ] 적용 대상:
  - 우측 상세 패널 (GraphPage, ImpactPage)
  - AddEdgeModal, SplitModal 등 모달 컨테이너
  - FloatingActionBar (ReviewPage)
  - Edge 검토 카드 리스트 컨테이너
  - ImpactPage 좌측 패널
  - Sidebar 컨테이너
- [ ] **적용 제외:** `ForceGraph2D` 캔버스 배경 (`#0f1117` 고정)
- [ ] 배경 블러가 활성화되려면 부모 요소에 투명 배경이 있어야 함 — AppShell 페이지 배경에 그라데이션 메시 추가:
  ```tsx
  <div className="fixed inset-0 bg-gradient-to-br from-slate-100 via-blue-50 to-indigo-100 -z-10" />
  ```

### Step 3 — 시스템 다크 모드 (FR-13.3)

- [ ] `tailwind.config.js`에 `darkMode: 'media'` 설정 (시스템 설정 자동 감지)
- [ ] 전체 다크 모드 색상 매핑:

  | 요소 | light | dark |
  |------|-------|------|
  | 페이지 배경 그라데이션 | `from-slate-100 via-blue-50 to-indigo-100` | `from-slate-900 via-slate-800 to-indigo-950` |
  | `.glass-panel` 배경 | `bg-white/70` | `dark:bg-slate-800/70` |
  | `.glass-panel` 보더 | `border-white/30` | `dark:border-slate-600/30` |
  | 카드/패널 텍스트 기본 | `text-slate-700` | `dark:text-slate-200` |
  | 카드/패널 텍스트 보조 | `text-slate-500` | `dark:text-slate-400` |
  | 인풋 배경 | `bg-white` | `dark:bg-slate-700` |
  | 인풋 보더 | `border-slate-200` | `dark:border-slate-600` |
  | Sidebar 배경 | glass-panel | (동일, 다크 매핑 적용) |

- [ ] **캔버스 배경 제외:** `ForceGraph2D` backgroundColor는 `#0f1117` 고정 (다크/라이트 무관)
- [ ] 필 배지: 배경색이 docColor이므로 다크 모드 별도 처리 불필요. 텍스트만 `text-white` 고정.

### Step 4 — 마이크로 애니메이션 (DOM) (FR-13.4)

- [ ] `tailwind.config.js`에 커스텀 keyframe 추가:
  ```js
  keyframes: {
    shimmer: { '0%': {backgroundPosition: '-200% 0'}, '100%': {backgroundPosition: '200% 0'} },
    'float-up': { '0%,100%': {transform: 'translateY(0)'}, '50%': {transform: 'translateY(-2px)'} },
    'impact-pulse': { '0%,100%': {boxShadow: '0 0 0 0 rgba(239,68,68,0.4)'}, '50%': {boxShadow: '0 0 0 6px rgba(239,68,68,0)'} },
  },
  animation: {
    shimmer: 'shimmer 2s linear infinite',
    'float-up': 'float-up 3s ease-in-out infinite',
    'impact-pulse': 'impact-pulse 2s ease-in-out infinite',
  }
  ```
- [ ] 요구사항 카드, Edge 카드에 hover 부상 적용:
  ```tsx
  className="... hover:-translate-y-1 hover:shadow-xl transition-all duration-200"
  ```
- [ ] 버튼(primary, secondary) hover에 `transition-all duration-150` 적용
- [ ] ReviewPage/GraphPage 패널 슬라이드 인: `transition-transform duration-300 ease-out`
- [ ] ImpactPage `affected` 항목 카드: `animate-impact-pulse` 클래스 추가

### Step 5 — Canvas 노드 hover 애니메이션 (FR-13.5)

> F26 Step 3 `nodeCanvasObject`에 추가되는 항목. F26 구현 시 함께 반영한다.

- [ ] `drawNode(ctx, node, globalScale, hoveredNodeId)` 함수에서 `node.id === hoveredNodeId` 일 때:
  - 기존: `scale 1.2×` + 흰색 외곽선
  - 추가: `ctx.shadowBlur = 20` (기본 대비 1.5×, 기본값 12 → hover 시 18~20)
  - `ctx.shadowColor = 'rgba(255,255,255,0.6)'`로 강화
- [ ] hover 상태는 `ForceGraph2D`의 `onNodeHover` 콜백에서 `hoveredNodeId` state로 관리
- [ ] 비-hover 노드는 기본 shadowBlur(8) 유지

### Step 6 — Skeleton shimmer + Framer Motion 트랜지션 (FR-13.6)

- [ ] 패키지 설치: `framer-motion`
- [ ] `Skeleton.tsx` 컴포넌트 교체: 단색 배경 → shimmer 그라데이션:
  ```tsx
  <div className="animate-shimmer bg-gradient-to-r from-slate-200 via-slate-100 to-slate-200
    bg-[length:200%_100%] rounded" style={{height, width}} />
  ```
- [ ] ReviewPage 요구사항 카드 리스트에 Framer Motion `layout` prop 적용:
  ```tsx
  <motion.div layout key={req.id} initial={{opacity:0, y:8}} animate={{opacity:1, y:0}}>
  ```
- [ ] ImpactPage 영향 결과 항목에 stagger 적용:
  ```tsx
  <motion.div
    initial={{opacity:0, y:12}} animate={{opacity:1, y:0}}
    transition={{delay: index * 0.08, duration: 0.25}}
  >
  ```
- [ ] AddEdgeModal / SplitModal 등장 애니메이션:
  ```tsx
  <motion.div initial={{opacity:0, y:16}} animate={{opacity:1, y:0}}
    transition={{type:'spring', stiffness:400, damping:30}}>
  ```
- [ ] F13 Skeleton 컴포넌트(기존 스피너/단색 박스)를 위 shimmer로 대체

### 타입 체크

- [ ] `npx tsc --noEmit` 오류 없음

## 주의사항

- Framer Motion `layout` prop은 리스트 항목의 key가 안정적이어야 한다(ID 기반). index key 사용 금지.
- `darkMode: 'media'`는 사용자 시스템 설정만 따른다. `localStorage` 저장 없음. 토글 UI 구현 금지.
- Glassmorphism `backdrop-blur`는 `isolation: isolate` 또는 `transform` 속성이 없는 조상 요소에서만 정상 동작한다. z-index 스태킹 컨텍스트 충돌 확인 필수.
- Pretendard Variable CDN은 `woff2` subset 방식이므로 초기 로드 시 FOUT(Flash Of Unstyled Text)가 발생할 수 있다. `font-display: swap`은 CDN에서 기본 적용되어 있음.
- Canvas 노드 shadowBlur는 GPU 부하가 있다. hover 상태 노드 1개에만 강화 적용하고 전체 노드에 shadowBlur를 남용하지 않는다.
