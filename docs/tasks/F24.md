# F24 — 그래프 가독성 개선 (Edge 간소화 / Semantic Zoom / 범례)

## 개요

노드 수 증가에 따른 그래프 가독성 저하를 해소하는 2가지 개선을 한 Feature로 묶는다.

1. **Edge 라벨 간소화** — 기본 숨김 + 선 스타일 구분 + hover 툴팁
2. **범례 패널** — 좌측 하단 상시 노출, 동적 구성

> Semantic Zoom은 F25로 분리되었다.

---

## 완료 기준

### 1. Edge 라벨 간소화

**`RequirementGraph.tsx` / Edge 커스텀 컴포넌트**

- [ ] 모든 Edge에서 라벨 텍스트 기본 제거
- [ ] `depends_on`: 실선(`strokeDasharray` 없음) + 단방향 화살표 (현행 유지)
- [ ] `related_to`: 점선(`strokeDasharray: '5,4'`) + 양방향 화살표 (현행 유지)
- [ ] PENDING Edge: 기존 스타일(반투명/회색) 유지
- [ ] Edge hover 시 툴팁 표시:
  - 내용: 관계 유형(`depends_on` / `related_to`) + evidence 문장 + confidence 수치(소수점 1자리, 예: `0.8`)
  - 구현: React Flow 커스텀 Edge 컴포넌트 내 `<EdgeLabelRenderer>`로 절대 위치 툴팁 렌더링
  - 툴팁 스타일: `bg-slate-900 text-white text-xs rounded-lg px-3 py-2 max-w-[240px] shadow-lg`

### 2. 범례 패널 (`GraphLegend.tsx` 신규)

- [ ] 위치: 그래프 영역 좌측 하단 (`position: absolute; bottom: 12px; left: 12px`)
- [ ] 기본 펼침 상태. 헤더("▾ 범례") 클릭으로 접기/펼치기
- [ ] 범례 항목 — Edge 선 스타일:
  - `───` depends_on (실선)
  - `┄┄┄` related_to (점선)
  - `- - -` PENDING (반투명 회색)
- [ ] 범례 항목 — 문서 색상:
  - 현재 그래프에 표시 중인 문서만 동적으로 렌더링
  - `● {filename}` 형식, `docColors` 팔레트 색상 적용
- [ ] ImpactPage에서 사용 시 영향 상태 범례 추가:
  - 🟠 changed
  - 🔴 affected
  - 🟡 review_recommended
- [ ] `GraphPage`와 `ImpactPage` 모두에서 사용 가능하도록 `showImpactLegend?: boolean` prop으로 분기
