# F24 — 그래프 가독성 개선 (Edge 간소화 / Semantic Zoom / 범례)

## 개요

노드 수 증가에 따른 그래프 가독성 저하를 해소하는 3가지 개선을 한 Feature로 묶는다.

1. **Edge 라벨 간소화** — 기본 숨김 + 선 스타일 구분 + hover 툴팁
2. **Semantic Zoom** — 줌 레벨별 노드/Edge 렌더링 디테일 조절
3. **범례 패널** — 좌측 하단 상시 노출, 동적 구성

---

## 완료 기준

### 1. Edge 라벨 간소화

**`RequirementGraph.tsx` / Edge 커스텀 컴포넌트**

- [ ] 모든 Edge에서 라벨 텍스트 기본 제거
- [ ] `depends_on`: 실선(`strokeDasharray` 없음) + 단방향 화살표 (현행 유지)
- [ ] `related_to`: 점선(`strokeDasharray: '5,4'`) + 양방향 화살표 (현행 유지)
- [ ] PENDING Edge: 기존 스타일(반투명/회색) 유지
- [ ] Edge hover 시 툴팁 표시:
  - 내용: 관계 유형(`depends_on` / `related_to`) + evidence 문장 + confidence 수치(소수점 1자리, 예: `0.8`)
  - 구현: React Flow 커스텀 Edge 컴포넌트 내 `<EdgeLabelRenderer>`로 절대 위치 툴팁 렌더링
  - 툴팁 스타일: `bg-slate-900 text-white text-xs rounded-lg px-3 py-2 max-w-[240px] shadow-lg`

### 2. Semantic Zoom

**`graphStore.ts`**

- [ ] `zoomLevel: number` 상태 추가 (초기값: 1)
- [ ] `setZoomLevel: (z: number) => void` 액션 추가
- [ ] debounce 적용 권장 (16ms — 60fps 기준)

**`RequirementGraph.tsx`**

- [ ] `onViewportChange` 콜백에서 `viewport.zoom`을 `setZoomLevel`에 전달

**`RequirementNode.tsx`**

- [ ] `zoomLevel`을 `graphStore`에서 구독
- [ ] 줌 레벨별 렌더링 분기:

  | 조건 | 노드 표시 |
  |------|-----------|
  | zoom < 0.6 | 색상 도트 + ID 라벨만 (제목 숨김, 연결 수 숨김) |
  | 0.6 ≤ zoom ≤ 1.2 | 라벨 + 제목 1줄 (`line-clamp-1`) |
  | zoom > 1.2 | 라벨 + 제목 2줄 (`line-clamp-2`) + 연결 수 뱃지 |

**Edge 커스텀 컴포넌트**

- [ ] zoom < 0.6 시 Edge 단순화: `strokeDasharray` 제거, 두께 균일(`strokeWidth: 1`), 색상만 유지
- [ ] zoom ≥ 0.6 복귀 시 원래 스타일 복원

### 3. 범례 패널 (`GraphLegend.tsx` 신규)

- [ ] 위치: 그래프 영역 좌측 하단 (`position: absolute; bottom: 12px; left: 12px`)
- [ ] 기본 펼침 상태. 헤더("▾ 범례") 클릭으로 접기/펼치기
- [ ] 범례 항목 — Edge 선 스타일:
  - `───` depends_on (실선)
  - `┄┄┄` related_to (점선)
  - `- - -` PENDING (반투명 회색)
- [ ] 범례 항목 — 문서 색상:
  - 현재 그래프에 표시 중인 문서만 동적으로 렌더링
  - `● {filename}` 형식, `docColors` 팔레트 색상 적용
- [ ] ImpactPage에서 사용 시 영향 상태 범례 추가:
  - 🟠 changed
  - 🔴 affected
  - 🟡 review_recommended
- [ ] `GraphPage`와 `ImpactPage` 모두에서 사용 가능하도록 `showImpactLegend?: boolean` prop으로 분기

## 주의사항

- F23(`RequirementNode` 디자인 개선)과 같은 파일을 수정하므로 F23 완료 후 또는 동시에 진행할 것
- Semantic Zoom의 `zoomLevel` 구독이 매 프레임 리렌더를 유발하지 않도록 debounce 또는 `useShallow` 적용 필요
- Edge hover 툴팁의 z-index가 노드보다 높아야 한다 (`z-50` 이상)
