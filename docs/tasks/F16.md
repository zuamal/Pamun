# F16 — Premium Motion & Interaction

## 개요

framer-motion 기반의 모션 레이어를 F14에서 구현한 컴포넌트 위에 입힌다. 데이터 생성·변경·소멸 과정을 물리적 움직임으로 시각화하여 인지 부하를 줄이는 것이 목표. B2B 생산성 툴에 맞게 기능과 반응성에 집중하고 불필요한 화려함은 배제한다.

**핵심 라이브러리:** `framer-motion`

F14(페이지 컴포넌트 구현)와 F13(Tailwind 설정)이 완료된 이후 진행한다.

## 완료 기준

### 1. ReviewPage — Layout 모션 (FR-9.1)

- [ ] `framer-motion` 패키지 설치 (`package.json`에 추가)
- [ ] `RequirementItem`을 `<motion.div layout>` 으로 교체
  - `layoutId`를 requirement ID로 설정하여 카드 위치 변경 시 FLIP 애니메이션 자동 적용
- [ ] 병합 시: 사라지는 두 카드에 `exit` 애니메이션 (opacity 0 + scale 0.95, 0.2초), 나머지 카드들이 `layout` prop으로 부드럽게 슬라이드 업
- [ ] 분할 시: 새로 생성된 카드에 `initial={{ opacity: 0, y: -8 }}` → `animate={{ opacity: 1, y: 0 }}` (0.25초), 하단 카드들이 `layout` prop으로 자연스럽게 밀려남
- [ ] `RequirementList`를 `<AnimatePresence>` 로 감싸 카드 exit 애니메이션 활성화
- [ ] `npx tsc --noEmit` 오류 없음

### 2. LLM 대기 상태 피드백 (FR-9.2)

- [ ] **Mesh Gradient Glow:** `ProgressModal` (F12) 또는 SSE 대기 중 표시되는 Skeleton 영역 배경에 맥동하는 그라데이션 글로우 추가
  - CSS `@keyframes` 또는 framer-motion `animate` + `transition.repeat: Infinity`로 구현
  - 예: `bg-gradient-to-r from-indigo-500/10 via-purple-500/10 to-indigo-500/10` + `animate-pulse` 또는 커스텀 keyframe
- [ ] **메시지 Fade 전환:** SSE `ProgressEvent.message` 변경 시 텍스트가 즉시 교체되지 않고 Fade-out(0.15초) → Fade-in(0.15초) 으로 전환
  - `<AnimatePresence mode="wait">` + `<motion.p key={message}>` 로 구현
- [ ] `npx tsc --noEmit` 오류 없음

### 3. GraphPage — 노드 포커싱 트랜지션 (FR-9.3)

- [ ] 노드 클릭 시 React Flow 인스턴스의 `fitView` 또는 `setCenter` 메서드를 duration과 함께 호출하여 해당 노드를 부드럽게 중앙으로 이동
  ```ts
  // NodeDetailPanel 또는 RequirementGraph 내부
  reactFlowInstance.fitView({
    nodes: [{ id: selectedNodeId }],
    duration: 800,
    padding: 0.5,
  })
  ```
- [ ] 미니맵 클릭 시에도 동일한 패닝 애니메이션 적용 (React Flow `<MiniMap onNodeClick>` 핸들러에서 호출)
- [ ] 급격한 시점 전환 없이 Ease-out 곡선으로 감속하며 이동
- [ ] `npx tsc --noEmit` 오류 없음

### 4. ImpactPage — Stagger & 하이라이트 애니메이션 (FR-9.4)

- [ ] **결과 항목 Stagger:**
  - `ImpactResultPanel`의 항목 목록을 `<motion.ul variants={containerVariants}>` 로 감싸고 각 `<ImpactItem>`을 `<motion.li variants={itemVariants}>` 로 교체
  - `containerVariants`: `staggerChildren: 0.08`
  - `itemVariants`: `initial: { opacity: 0, y: 12 }` → `animate: { opacity: 1, y: 0 }` (duration 0.25초)
  - 영향 분석 결과가 새로 갱신될 때마다 Stagger 재실행 (`key` prop 활용)
- [ ] **하이라이트 Width 애니메이션:**
  - `DocumentViewer`의 하이라이트 `<span>` 배경을 CSS `width` 또는 `scaleX` 트랜지션으로 구현
  - 초기 상태 `width: 0` → `width: 100%` (0.3초, ease-out)
  - framer-motion `<motion.span>` 또는 Tailwind `transition-[width]` + JS로 클래스 토글
- [ ] `npx tsc --noEmit` 오류 없음

### 5. Glassmorphism + Spring 모달 (FR-9.5)

- [ ] **Glassmorphism 스타일:** `AddEdgeModal`, `SplitModal`, `FloatingActionBar`에 적용
  - 배경: `bg-white/80 backdrop-blur-md border border-white/20`
  - 뒤쪽 콘텐츠(그래프, 텍스트)가 흐릿하게 비쳐 맥락 유지
- [ ] **Spring 등장 애니메이션:** 세 컴포넌트 공통
  ```ts
  initial: { opacity: 0, y: 24, scale: 0.97 }
  animate: { opacity: 1, y: 0, scale: 1 }
  transition: { type: "spring", stiffness: 300, damping: 28 }
  ```
- [ ] `FloatingActionBar`의 슬라이드 업 (`y: 64 → y: 0`) 도 Spring으로 교체
- [ ] `<AnimatePresence>` 로 감싸 닫힐 때 exit 애니메이션 (`opacity: 0, y: 16`, 0.15초) 적용
- [ ] `npx tsc --noEmit` 오류 없음

### 6. GraphPage — 매그네틱 엣지 스냅 강조 (FR-9.6)

- [ ] React Flow `connectionRadius` prop을 설정하여 핸들 근방 스냅 반경 정의 (예: `40px`)

## 구현 시 주의사항

- `layout` prop은 부모 컴포넌트에 `<LayoutGroup>` 이 필요할 수 있다. `RequirementList` 전체를 `<LayoutGroup>` 으로 감싼다.
- Stagger 재실행은 `key={JSON.stringify(impactResult)}` 같은 방식으로 `motion.ul` 의 key를 변경하여 트리거한다.
- `fitView`는 React Flow 17+ 기준 `useReactFlow()` 훅의 인스턴스 메서드로 호출한다.
- Glassmorphism은 `backdrop-filter` 지원 브라우저(Chrome 76+, Safari 9+)에서만 동작한다. 미지원 환경에서는 불투명 배경으로 graceful degradation.
- framer-motion 번들 크기: ~45KB gzip. 포트폴리오 규모에서 허용 범위.
