# F26 — Force-Directed WebGL 그래프 대개편

## 개요

React Flow + Dagre 기반 그래프를 `react-force-graph-2d`(WebGL/Canvas + d3-force)로 전환한다.
GraphPage와 ImpactPage 모두 교체한다. F23(노드 디자인), F24(Edge 간소화/범례), F25(Semantic Zoom)를 본 Feature로 통합하여 대체한다.

> **브랜치:** `exp/force-graph` 등 실험 브랜치에서 진행 후 main에 머지

---

## 완료 기준

### Step 1 — 라이브러리 교체

- [ ] 패키지 제거: `@xyflow/react`, `@dagrejs/dagre`
- [ ] 패키지 설치: `react-force-graph-2d`, `d3-force`
- [ ] 타입 패키지 설치: `@types/react-force-graph` (필요 시)

### Step 2 — 데이터 어댑터 (`src/lib/graphAdapter.ts` 신규)

- [ ] `Requirement[]` + `Edge[]` → `{ nodes: GraphNode[], links: GraphLink[] }` 변환
  ```ts
  interface GraphNode {
    id: string
    label: string        // 요구사항 ID (예: PRD-1)
    title: string        // 요약 제목
    docId: string        // 문서 ID (색상 결정용)
    color: string        // docColors 팔레트 색상
    approvedCount: number // APPROVED Edge 수 (연결 수 뱃지용)
    status?: 'changed' | 'affected' | 'review_recommended'
    pinned?: boolean
    fx?: number          // 고정 위치 x
    fy?: number          // 고정 위치 y
  }
  interface GraphLink {
    id: string
    source: string
    target: string
    relationType: 'depends_on' | 'related_to'
    status: 'approved' | 'pending' | 'rejected'
    evidence: string
    confidence: number
  }
  ```

### Step 3 — Canvas 노드 커스텀 렌더링 (`nodeCanvasObject`)

- [ ] Canvas 2D API로 노드를 직접 그리는 `drawNode(ctx, node, globalScale)` 함수 구현:

  | globalScale (카메라 줌) | 렌더링 내용 |
  |------------------------|-------------|
  | ≤ 0.3 | 색상 도트만 (원형, r=6) |
  | 0.3 < scale < 0.9 | 도트 + ID 라벨 (`ctx.fillText`) |
  | 0.9 – 1.2 | 라벨 + 제목 1줄 |
  | > 1.2 | 라벨 + 제목 2줄 + 연결 수 뱃지 |

- [ ] Impact 상태 시각화:
  - `changed`: 주황 외곽 글로우 (`ctx.shadowBlur`, `ctx.shadowColor`)
  - `affected`: 빨간 링 (`ctx.strokeStyle`)
  - `review_recommended`: 노란 링
- [ ] hover 상태: 노드 스케일 1.2×, 흰색 외곽선 추가

### Step 4 — Edge 렌더링

- [ ] `depends_on`: 실선 + 단방향 화살표
- [ ] `related_to`: 점선(`lineDash`) + 양방향 화살표
- [ ] PENDING: 반투명 회색
- [ ] Edge 두께 보정: `strokeWidth = max(2, 기본값 / globalScale)` (zoom < 0.9 구간)
- [ ] 화살표 마커는 strokeWidth에 자동 비례 (별도 보정 불필요)
- [ ] Edge hover 시 툴팁(DOM 오버레이): 관계 유형 + evidence + confidence

### Step 5 — d3-force 물리 파라미터

- [ ] `forceLink`: `depends_on` distance=120, `related_to` distance=80
- [ ] `forceManyBody`: strength=-300 (강한 척력, 노드 겹침 방지)
- [ ] `forceCenter`: 캔버스 중앙 gravity
- [ ] 시뮬레이션 수렴 후(`d3AlphaDecay`) 자동 정지

### Step 6 — 인터랙션

**노드 클릭:**
- [ ] 일반 클릭: 우측 패널에 노드 상세 표시 (기존 동작 유지)
- [ ] 수동 Edge 추가 (클릭-투-커넥트):
  - source 노드 클릭 → `pendingSource` 상태 설정, 노드 강조
  - 다른 노드 클릭 → Edge 생성 모달 오픈 (`AddEdgeModal` 재사용)
  - ESC 또는 빈 캔버스 클릭 → `pendingSource` 초기화

**노드 드래그:**
- [ ] 드래그 완료 시 `node.fx = x`, `node.fy = y`로 위치 고정(pin)
- [ ] 고정된 노드를 다시 클릭하면 pin 해제 (`fx = undefined`)

**포커스 모드 (Neighborhood Ghosting):**
- [ ] 노드 hover 시 1-hop 연결 노드/Edge만 opacity 1.0 유지
- [ ] 나머지 모든 노드 opacity 0.1로 페이드
- [ ] hover 해제 시 전체 복원

### Step 7 — Edge 검토 카드 ↔ 그래프 연동 (F24 인수)

- [ ] 우측 Edge 검토 카드 hover → `hoveredEdgeId` 설정 → 해당 Link 강조 (두께 + 색상)
- [ ] `ForceGraph2D`의 `linkCanvasObjectMode` 또는 `linkColor`/`linkWidth` 콜백에서 분기

### Step 8 — 파티클 애니메이션 (ImpactPage)

- [ ] ImpactPage 그래프에서 `changed` 노드 존재 시 `linkDirectionalParticles` 활성화
- [ ] 파티클 방향: `depends_on`은 source→target, `related_to`는 양방향
- [ ] 파티클 색상: 주황(changed 방향), 빨간(affected 방향)
- [ ] ImpactPage에서만 적용. GraphPage에서는 파티클 비활성화

### Step 9 — 범례 패널 (F24 인수, DOM 오버레이)

- [ ] `GraphLegend.tsx` 신규. Canvas 위에 `position: absolute` DOM 오버레이로 배치
- [ ] 위치: 좌측 하단
- [ ] 기본 펼침. 현재 그래프 문서 목록 + Edge 선 스타일 동적 표시
- [ ] ImpactPage: `showImpactLegend={true}`로 영향 분석 상태 범례 추가

### Step 10 — 다크 코스모스 테마

- [ ] 캔버스 배경: `#0f1117` (deep dark)
- [ ] 노드 기본 색상: 문서별 `docColors` 팔레트 유지, 불투명도 0.9
- [ ] 노드 내부 텍스트: `#e2e8f0` (slate-200)
- [ ] Edge 기본 색상: `rgba(148, 163, 184, 0.4)` (반투명 slate-400)
- [ ] 선택/hover 노드: 흰색 외곽 글로우

### Step 11 — Zustand 연동

- [ ] `graphStore`에서 `zoomLevel` 제거 (Canvas `globalScale`로 대체)
- [ ] `pendingSource: string | null` 추가 (클릭-투-커넥트 상태)
- [ ] `pinnedNodes: Record<string, {x: number, y: number}>` 추가

## 주의사항

- `nodeCanvasObject`를 사용하면 React Flow처럼 HTML 엘리먼트를 노드 내부에 넣을 수 없다. 모든 노드 UI는 Canvas 2D API로 그려야 한다.
- `react-force-graph-2d`는 내부적으로 `canvas` 엘리먼트를 관리하므로, 클릭 좌표는 Canvas 이벤트 콜백(`onNodeClick`, `onLinkClick`)으로만 처리한다. DOM 이벤트 버블링 없음.
- 다크 테마는 AppShell의 사이드바/패널과 색상 일관성을 맞춰야 한다. Canvas 배경만 어둡고 우측 패널이 밝으면 어색하다 — GraphPage/ImpactPage 전체 배경색 조정 필요.
- 물리 시뮬레이션 수렴 시간(보통 2–5초) 동안 노드가 움직인다. 초기 로드 시 Skeleton이나 로딩 메시지를 표시하는 것을 권장한다.
