# F21 — 데모 워크플로우 자연스러운 흐름 개선

## 개요

현재 데모 번들 선택 시 즉시 `/graph`로 점프하는 구조를 변경한다.
LLM 호출 단계만 fake progress로 대체하고, 나머지 모든 단계는 실제 사용 흐름과 동일하게 유지한다.

**변경 전:**
```
번들 선택 → store 전체 초기화 → /graph 이동
```

**변경 후:**
```
번들 선택 → 문서 표시(UploadPage)
→ "파싱 시작" (fake progress ~2초) → ReviewPage
→ "의존관계 추론 시작" (fake progress ~2초) → GraphPage
→ ImpactPage
```

## 주의사항

- fake progress의 타이밍(ms)은 UX 판단에 따라 DEV가 조정 가능. 너무 빠르면 진행이 체감되지 않고, 너무 느리면 지루하다. 2–3초 내외를 권장한다.
- `demoStore.pendingBundle`은 번들 로드 시 1회 채워지고, 이후 파싱/추론 시뮬레이션에서 소비된다. 세션 리셋 시 함께 초기화한다.
- 기존 ProgressModal 컴포넌트를 그대로 재사용한다. fake progress도 동일한 UI로 표시된다.
